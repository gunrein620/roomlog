# T-DEF nav-manifest — 화면×클릭요소×목적지 대조표

> 소스: `roomlog_screens_tenant-defect.md` §(3) 전이 테이블 + `T-DEF 하자.dc.html`(와이어 실측, `onClick` 핸들러 기준)
> 라우트 상수: `lib/nav.ts`
> 표기: 종류 = **route**(페이지 이동, next/link로 감쌀 대상) / **in-screen**(같은 화면 내 상태 변화, 라우팅 없음) / **system**(사용자 클릭 없는 자동 전이 — 로딩 완료·타임아웃 등)

## 대조 원칙(중요 — builder 참고)
- 스펙 전이표 1행이 와이어에서 **서로 다른 라벨의 별개 DOM 요소 2개**로 구현된 경우, 이 표에서는 **행을 분리**했다(목적지 없는 버튼 방지). 아래 4곳:
  1. T-DEF-00 "진행 중 카드 / 알림 벨" → 카드 탭 · 벨 버튼 별개
  2. T-DEF-02 "뒤로" → 진입 모드에 따라 목적지 다름(신규=01 / 추가정보·재촬영=11)
  3. T-DEF-04 "관리자 검토 요청(이의/판단어려움)" → 임차인책임·이의 컨텍스트는 라벨 "이의 있음 · 관리자 검토 요청", 판단어려움 컨텍스트는 라벨 "관리자 검토 요청"(별개 버튼)
  4. T-DEF-10 "재요청 / 기록 보기" → "재요청" 버튼과 "전·후 사진 / 기록 보기" 버튼 별개
- T-DEF-09 "관리자와 채팅"은 진입 사유(대기/확정·동의미완/동의완료·업체확보) 3가지 컨텍스트에 모두 등장하지만 목적지가 항상 T-DEF-11로 동일 → **분리하지 않고 1행 유지**(조건부 목적지가 아니라 조건부 노출일 뿐).
- system 전이(3건: 03→04, 03→E0, 09→10)는 route 카운트에서 **제외**. 사용자가 누르는 버튼이 아니라 자동 전이이므로 별도 처리(로딩 타임아웃/데모 트리거)가 필요.
- **와이어의 "WF 시나리오" 탭류(신규/추가정보/재촬영, 임차인/임대인/판단어려움, 위험 토글, 진입사유·결정상태 탭, 임차인책임/임대인책임 컨텍스트 탭, 빈상태·수리단계 토글)는 Claude Design 데모용 상태 시뮬레이터이지 프로덕션 UI가 아니다 — 이 표에서 제외. route로 오해해 배선하지 말 것.**

## 이상값 / 예외 (평균에 묻히지 않게 별도 명시)
- **도메인 카드의 "02·04 뒤로 조건부" 언급 중 04는 스펙과 불일치**: `roomlog_screens_tenant-defect.md`의 T-DEF-04 카드·전이표 모두 뒤로 목적지가 `T-DEF-00` 단일이다(조건 분기 없음). 조건부 뒤로는 **02만 해당**(신규=01 / 추가정보·재촬영=11). 카드 메모가 구버전 가정을 담고 있을 가능성 — 정본(전이표) 기준으로 채택.
- **T-DEF-06 "취소/환불 안내"·T-DEF-11 "재요청 제출"**: 스펙 화면카드에는 명시돼 있으나 현재 `T-DEF 하자.dc.html`에는 대응 DOM 요소가 없다. 셸 슬라이스 목적(버튼→라우팅만) 범위에서는 실제 없는 버튼을 만들 수 없으므로 이번 배선 대상에서 제외하고 위 표에 "미시각화"로 기록. 와이어 갱신 시 재확인 필요.
- **T-DEF-05 동의 체크박스**·**T-DEF-10 만족도 선택**: 스펙의 "인-스크린(상태 변화)" 요약 섹션에는 각각 명시적으로 나열돼 있지 않지만, 와이어에는 실제 인터랙션(체크 토글) 또는 정적 자리표시자(만족도, onClick 없음)로 존재 — 추가 발견으로 포함.
- **WF 데모 탭류 제외**: 02(신규/추가정보/재촬영 전환 탭), 04(임차인/임대인/판단어려움·위험 시나리오 탭), 09(진입사유 4탭·결정상태 3탭), 11(임차인책임/임대인책임 컨텍스트 탭), 00·08의 상태 토글 버튼 — 전부 `data-drags-parent`/"WF 시나리오" 주석이 붙은 프리뷰 전용 컨트롤이며 스펙에도 사용자 대면 요소로 정의돼 있지 않다. route 아님, 배선 대상 아님.

## 한계 / 불확실성
- 이 표는 `T-DEF 하자.dc.html`(현재 버전)과 스펙 v3를 대조한 결과다. 와이어가 갱신되면(특히 위 "미시각화" 2건이 추가되면) 재대조 필요.
- system 전이 3건(03→04, 03→E0, 09→10)은 이 슬라이스에서 버튼 배선 대상이 아니므로 shell-builder가 데모용 트리거(예: setTimeout 또는 개발용 버튼)로 처리할지 팀장 확인 필요.
