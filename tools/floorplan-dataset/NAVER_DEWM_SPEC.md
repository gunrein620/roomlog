## 2. 워터마크 제거 v2 — 알고리즘 요구사항
새 스크립트(예: `dewatermark_naver_v2.py`) 작성. 핵심 원칙 두 가지:
**(A) inpaint 절대 사용 금지. 제거는 반투명 역산(alpha subtraction)만.**
**(B) 변형에 무관하게 이미지별로 워터마크를 직접 검출(글로벌 고정 마스크 금지).**

이미지별 처리:
1. 중앙 밴드로 한정(923×676 기준 대략 y∈[230,445], x∈[225,700]; 다른 크기는 비율 스케일 또는 원본 복사).
2. 국소 배경 B 추정: 큰 커널 median blur 등(얇은 글자는 사라지고 바닥/벽 색만 남게).
3. **워터마크 후보 픽셀** = 밴드 내에서 "B보다 밝고(value↑) 채도는 낮은(saturation↓)" 픽셀.
   - 반투명 흰 글자가 채색 바닥 위에 얹힌 특성. → 흰 벽 위에선 상승폭이 작아 자연 배제, **어두운 벽선은 더 밝지 않으므로 배제(벽 보존)**.
4. **alpha 맵 추정**: 흰색 오버레이 I=(1−α)J+αC, C≈흰색. value 상승분 lift=V_I−V_B ≈ α(255−V_B) → α=lift/(255−V_B), [0, ~0.6] 클램프. 가장자리 약한 가우시안 페더링.
5. **역산 복원**: J=(I−αC)/(1−α), 채널별. **inpaint 없음.** 어두운 벽선은 α≈0이라 값이 그대로 유지 → 벽 온전.
6. C(워터마크 색)는 near-white 고정 또는 이미지별 소폭 보정 허용.

목표: 3종 변형 모두 실제 위치에 맞춰 걷어내되, **어떤 경우에도 벽선·라벨의 어두운 픽셀을 지우거나 끊지 않음.**
