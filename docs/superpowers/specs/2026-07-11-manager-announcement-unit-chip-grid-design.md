# 관리자 공지 호실 칩 그리드 설계

## 목표

`/manager/messaging/01`의 호실 선택 목록에서 브라우저 기본 체크박스를 제거하고, 호실 번호 카드 전체를 누르는 간결한 칩 그리드로 변경한다.

## 선택안

비주얼 비교에서 A안인 `호실 번호 칩 그리드`를 채택한다. 많은 호실을 한눈에 보고 빠르게 복수 선택하기 위한 방식이다.

## 범위

- 수정 화면: `/manager/messaging/01`
- 수정 대상: 호실 목록 마크업, CSS Module, 관련 web 계약 테스트
- 제외 대상: 건물 드롭다운, 대상 계산, 저장 payload, API, `/manager/messaging/00`, `/02`, 인프라 설정

## UI 구조

- 선택한 건물의 호실을 최대 5열 그리드로 표시한다.
- 화면 폭이 부족하면 CSS grid가 자동으로 열 수를 줄여 줄바꿈한다.
- 각 항목에는 건물명을 반복하지 않고 `101호`처럼 `roomNo`만 표시한다.
- `roomNo`가 없으면 기존 `roomDisplayLabel(room)`을 대체 문구로 사용한다.
- 카드 전체가 선택 영역이며 최소 터치 높이를 유지한다.

## 선택 상태

- 미선택 카드는 기본 표면 배경, 얇은 테두리, 기본 글자색을 사용한다.
- 선택 카드는 기본 색상 배경과 반전 글자색을 사용한다.
- 선택 카드 우측에는 `✓` 표시를 추가한다.
- 카드 클릭 시 기존 `toggleRoom(room.id)`로 복수 선택과 해제를 처리한다.

## 접근성

- 실제 `input type="checkbox"`는 유지하되 공용 숨김 입력 방식으로 시각적으로만 숨긴다.
- input과 시각 카드가 같은 label 안에 있어 카드 전체 클릭으로 상태가 바뀐다.
- 키보드 Tab으로 각 호실에 이동하고 Space로 선택할 수 있다.
- focus-visible 상태에는 공용 focus ring을 표시한다.
- 선택 여부는 `input:checked + chip` CSS 규칙으로 표현한다.

## 기존 기능 보존

- 호실 범위 진입과 건물 변경 시 선택 초기화 동작을 유지한다.
- 선택한 건물의 호실만 표시하는 `roomsForBuilding` 흐름을 유지한다.
- 복수 호실 대상 요약과 저장 payload를 변경하지 않는다.
- 선택 가능한 호실이 없는 경우 기존 빈 상태 문구를 유지한다.

## 테스트

- `/01` 소스 계약 테스트에서 숨겨진 checkbox와 호실 chip label 구조를 확인한다.
- 원래 보이는 기본 checkbox 스타일이 사용되지 않는지 확인한다.
- 선택 체크 표시와 `toggleRoom` 연결을 확인한다.
- CSS 계약에서 5열 자동 그리드, checked 상태, focus ring을 확인한다.
- web 단위 테스트와 프로덕션 빌드를 실행한다.
- Docker 브라우저에서 기본 checkbox가 보이지 않고 호실 카드 선택 시 파란 배경과 체크 표시가 나타나며 복수 선택되는지 확인한다.

## 완료 조건

- 호실 목록이 최대 5열의 카드형 칩 그리드로 표시된다.
- 기본 체크박스는 보이지 않지만 마우스, 터치와 키보드 선택이 유지된다.
- 선택 카드의 파란 배경, 흰 글자와 체크 표시가 명확하다.
- `/manager/messaging/01` 외 제품 화면과 인프라 파일은 변경되지 않는다.
