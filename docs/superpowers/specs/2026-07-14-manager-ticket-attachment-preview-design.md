# 관리인 민원 상세 첨부 이미지 미리보기 설계

## 목표

관리인 민원·하자 대시보드의 상세 모달에서 첨부 이미지 파일명만 표시하지 않고 실제 썸네일을 보여준다. 관리인은 썸네일을 선택해 같은 화면에서 큰 원본 미리보기를 확인할 수 있어야 한다.

## 범위

- 기존 `GET /manager/tickets` 응답의 `messages[].attachmentUrls`를 web 매핑 계층에서 보존한다.
- 티켓별 첨부 URL을 중복 없이 대시보드 행 데이터에 포함한다.
- 상세 모달의 상세 내용 아래에 첨부 이미지 썸네일 목록을 표시한다.
- 썸네일을 누르면 모달 위에 큰 이미지 오버레이를 표시한다.
- 큰 이미지 오버레이는 닫기 버튼, 바깥 영역 선택, `Escape` 키로 닫을 수 있다.
- 이미지 URL이 없는 과거 파일명 기록은 현재 텍스트 표시를 유지한다.
- 이미지 로딩 실패 시 깨진 이미지 아이콘 대신 파일명 링크를 제공한다.

## 제외 범위

- 첨부 이미지 업로드 방식 변경
- API 또는 데이터베이스 스키마 변경
- 이미지 편집, 삭제, 다운로드 전용 기능
- 비이미지 첨부의 문서 뷰어

## 데이터 흐름

1. 관리인 티켓 목록 응답에서 각 메시지의 `attachmentUrls`를 읽는다.
2. `ticket-manager-api`가 URL을 정리하고 중복을 제거해 `DefectDashboardRow.attachmentUrls`로 전달한다.
3. `TicketDetailDialog`가 URL별 썸네일과 접근 가능한 대체 텍스트를 렌더링한다.
4. 사용자가 썸네일을 선택하면 선택 URL을 상태로 보관하고 큰 이미지 오버레이를 연다.
5. 닫기 동작이 발생하면 선택 URL을 해제하고 상세 모달로 초점을 돌린다.

## UI 및 접근성

- 썸네일은 실제 이미지를 알아볼 수 있는 고정 크기 카드로 배치한다.
- 각 썸네일 버튼에는 파일명을 포함한 접근 가능한 이름을 제공한다.
- 큰 이미지 오버레이는 대화상자 의미와 닫기 버튼을 제공한다.
- `Escape`는 큰 이미지가 열려 있으면 큰 이미지만 먼저 닫는다.
- 스타일은 `packages/ui/src/tokens.css`에서 제공하는 CSS 변수만 사용하며 raw hex 값을 추가하지 않는다.

## 오류 처리

- 비어 있거나 중복된 URL은 렌더링 전에 제거한다.
- 이미지 로딩에 실패하면 해당 항목을 원본 URL로 이동할 수 있는 파일명 링크로 바꾼다.
- 첨부 URL이 없으면 별도 빈 영역을 만들지 않는다.

## 테스트 및 검증

- 실패 테스트로 티켓 메시지의 첨부 URL 매핑, 중복 제거, 상세 모달 썸네일·큰 이미지 동작 계약을 먼저 고정한다.
- 대상 web 테스트를 통과시킨다.
- `pnpm test:web`을 실행한다.
- `bash scripts/verify.sh`로 기본 통합 검증을 수행한다.
- 필요하면 Docker web 이미지를 재빌드하고 실제 대시보드 모달에서 썸네일과 큰 이미지 열기·닫기를 확인한다.

## 인프라 영향

없다. 기존 API 응답과 web 코드만 사용하며 compose, Dockerfile, workflow, 배포 환경 변수는 수정하지 않는다.
