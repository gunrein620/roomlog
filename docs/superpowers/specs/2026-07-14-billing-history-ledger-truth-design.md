# 청구수납 실적 기간·입출금 원장 진실성 개선 설계

## 목표

- 수금 실적 표는 실제 기록이 존재하는 범위 안에서 최근 최대 6개월을 보여주고 사용자가 3개월, 6개월, 12개월 또는 직접 지정한 월 범위로 바꿀 수 있게 한다.
- 표는 최근 월부터 보여주는 것을 기본값으로 하되 과거순으로 전환할 수 있게 한다.
- 입출금 내역에서 내부 ID와 프론트가 임의 생성한 값을 제거하고, API가 확인한 DB 데이터만 표시한다.
- 출금 명칭은 유지한다. 출금 행은 샘플이 아니라 DB의 확정 비용 기록만 사용한다.

## 범위

### 포함

- `수금 현황 > 수금 실적 변화`의 기간 선택, 정렬, 고정 높이 내부 스크롤
- 수금 실적 조회 범위를 API 쿼리로 전달하는 계약
- `입출금 내역` 전용 원장 응답 타입과 API 조립 로직
- 입금 상세의 사용자용 납부·청구 요약 표시
- API 미기동 데모 폴백의 명시적 표시

### 제외

- 새 DB 테이블, 스키마 변경, 마이그레이션
- 은행 출금 연동
- 입출금 등록·수정 UI
- 비용 도메인의 상태 정책 변경
- 청구수납 밖의 화면 개편

## 수금 실적 변화 UX

### 기간

- 빠른 선택: `3개월`, `6개월`, `12개월`, `직접 설정`
- 기본값: `최근 최대 6개월`. 기록이 2개월뿐이면 2개월만 표시한다.
- 빠른 선택과 직접 설정의 종료 월은 상단에서 선택한 청구월을 기준으로 한다.
- 직접 설정에서는 시작 월과 종료 월을 선택한다.
- 직접 설정 가능 범위는 선택 건물에 실제 청구서가 존재하는 최초 월부터 마지막 기록 월까지다.
- 최초 기록 월 이전과 마지막 기록 월 이후의 월을 0원 실적으로 생성하지 않는다.
- 청구 기록이 전혀 없으면 한 달짜리 0원 행을 만들지 않고 실적 없음 상태를 표시한다.
- 시작 월이 종료 월보다 늦으면 적용하지 않고 입력 가까이에 오류를 표시한다.
- 기간 선택은 실적 표의 조회 범위만 바꾸며, 상단의 최근 3개월·6개월 평균 산식은 그대로 유지한다.

### 정렬과 높이

- 기본 정렬: `최근순`
- 전환 옵션: `최근순`, `과거순`
- 행 영역은 6개 행 높이로 고정한다.
- 선택 결과가 6개를 넘으면 표 본문 내부만 세로 스크롤한다.
- 헤더는 스크롤 중에도 고정한다.
- 전월 대비 값은 화면 정렬과 무관하게 실제 월의 시간순으로 계산한다.

### URL과 조회

- 기간과 정렬은 URL 검색 파라미터에 보존해 새로고침과 뒤로 가기에서도 유지한다.
- 빠른 선택은 개월 수로, 직접 설정은 시작·종료 월로 표현한다.
- API는 요청받은 월 범위만 집계한다. 임의의 장기간 전체 이력을 최초 로드에 내려보내지 않는다.
- 응답에는 선택 건물의 조회 가능한 최초·마지막 청구월을 함께 포함한다.

## 입출금 원장 데이터

### 공통 원칙

- 프론트는 거래 의미를 추론하지 않는다.
- 원장 행에 필요한 표시 데이터는 API가 접근 권한을 확인한 뒤 조립한다.
- 내부 식별자는 행 key와 mutation에만 사용하고 화면 텍스트로 노출하지 않는다.
- DB에 없는 값은 `정보 없음` 또는 `미연결`로 표시한다. 대체 이름이나 날짜를 생성하지 않는다.

### 입금 행

입금 행은 `Deposit`을 기준으로 하고 연결된 `Bill`, `Room`, 임차인 정보를 API에서 결합한다.

- 입금자명, 입금액, 입금일, 매칭 상태: `Deposit`
- 건물, 호실: 연결된 청구서의 실제 호실. 미연결 입금의 `guessedUnitId`는 사실값처럼 섞지 않고 `연결 후보`로 구분한다.
- 임차인: 연결된 청구서·호실의 실제 임차인
- 납부 상세: 청구월, 납부기한, 청구 금액, 납부 금액, 청구 항목
- 미연결 입금: 상세 영역에 `청구 내역 확인 필요` 표시

건물·호실·임차인은 상위 거래 행에 이미 있으므로 상세에서 반복하지 않는다. 내부 관계명인 `연결 청구`와 `연결 후보 청구`는 사용자에게 표시하지 않는다.

청구 항목은 연결된 청구서의 `BillLineItem`을 그대로 사용하되 `청구 내역 월세 650,000원 · 관리비 70,000원`처럼 한 묶음으로 표시한다. 각 항목마다 같은 라벨을 반복하지 않는다. 납부기한은 ISO 시간 문자열을 노출하지 않고 서울 기준 한국식 날짜만 표시한다.

### 입금 펼침 상세의 정보 구조

입금 상세는 한 문장에 청구월과 항목을 섞지 않고 두 묶음으로 정렬한다.

- 입금 정보: 입금자, 입금일시, 이번 입금, 처리 상태
- 청구 정보: 청구월, 청구 내역, 납부기한, 청구금액, 누적 수납, 미수금

`이번 입금`은 선택한 원장 행의 거래 금액이고, `누적 수납`은 연결 청구서의 전체 확정 수납액이다. 미수금은 청구금액에서 누적 수납을 뺀 값이며 0원 아래로 내려가지 않는다. 처리 상태는 연결 관계와 청구 상태를 바탕으로 `완납`, `부분 수납`, `수납 대기`, `연결 확인 필요`처럼 사용자가 이해할 수 있는 말로 표시한다. 내부 ID, DB에 없는 계좌·결제수단, 추정 정보는 추가하지 않는다.

### 출금 행

- 방향 표시는 사용자가 확정한 대로 `출금`을 유지한다.
- 기존 `buildWithdrawalRows` 샘플 행은 삭제한다.
- DB의 `Cost` 중 관리자가 접근할 수 있고 상태가 `confirmed` 또는 `amended`인 기록만 출금 행으로 사용한다.
- 날짜, 항목, 금액, 비용 유형, 귀속 범위, 호실, 증빙 여부는 `Cost`와 연결 `Receipt`의 실제 값만 사용한다.
- 비용 데이터에 거래처나 건물이 기록되지 않았다면 생성하지 않고 `정보 없음`으로 표시한다.
- 이 출금은 은행 거래 연동 결과가 아니라 확정 비용 원장임을 상세 영역에 짧게 명시한다.

### 제거할 임시 로직

- 배열 인덱스로 건물을 만드는 `buildingFor`
- 금액·순서로 항목을 추정하는 `chargeKindFor`와 `splitAmount`
- 고정 임대기간 배열
- 화면에서 만드는 메모
- 샘플 출금 생성 로직
- 내부 청구서 ID의 직접 출력

## API와 공유 타입

- `packages/types`에 화면이 소비할 관리인 입출금 원장 행 타입을 추가한다.
- 입금 매칭 mutation에는 기존 deposit ID와 bill ID를 계속 사용하되 표시 DTO와 분리한다.
- 수금 분석 타입에 조회 가능 범위와 실제 적용 범위를 추가한다.
- 관리인 수금 API는 `historyFrom`, `historyTo`를 검증하고 관리 범위에 속한 청구서만 집계한다.
- 입출금 API는 입금·확정 비용을 하나의 정렬 가능한 표시 DTO로 반환한다.
- 기존 DB 모델만 사용하므로 마이그레이션은 만들지 않는다.

## 데모 폴백

AGENTS.md의 API 미기동 렌더 요구사항은 유지한다. 다만 실제 원장과 혼동하지 않도록 응답에 데이터 출처를 포함하고, 데모 폴백일 때 화면 상단에 `데모 데이터`를 명시한다.

## 오류와 빈 상태

- 유효하지 않은 월 형식 또는 역전된 범위는 API에서 거부하고 화면에서도 적용 전에 안내한다.
- 선택 범위에 실적이 없으면 0원짜리 월 행 대신 해당 기간에 청구 실적이 없다는 메시지를 표시한다.
- DB에 확정 비용이 없으면 출금 샘플을 생성하지 않는다.
- 연결 청구서가 없거나 관리자 범위 밖이면 내부 정보를 노출하지 않고 `미연결`로 처리한다.

## 월중 확정 수납 흐름 가독성

- 제목·범례 행과 하단 요약·차트 사이에 목록 구분선과 같은 색상·굵기의 1px 구분선을 둔다.
- 범례는 상대 표현인 `이번 달`, `지난달` 대신 실제 월을 표시한다.
- 날짜별 막대는 선택 청구월을 왼쪽 보라색, 직전 청구월을 오른쪽 중립색으로 고정한다.
- 날짜 축은 매일 작은 눈금을 표시하고 숫자는 1일, 5일 단위, 선택 월의 말일만 표시한다.
- 선택 월의 실제 일수만 렌더링해 2월은 28일 또는 29일, 4·6·9·11월은 30일에서 끝낸다.
- 각 날짜 막대에 마우스를 올리거나 키보드로 접근하면 날짜와 두 월의 누적 수납액을 확인할 수 있다.
- 직전 청구월의 누적 수납 기록이 전부 0원이면 보이지 않는 0 높이 막대만 두지 않고 해당 월의 수납 기록이 없음을 안내한다.
- 기존 `currentCumulativeAmount`, `previousCumulativeAmount`를 사용하므로 API·DB 계약은 변경하지 않는다.

## 테스트

- API: 사용자 지정 기간, 건물 범위, 최초·마지막 월, 최근순과 무관한 전월 대비 기반 데이터
- API: 입금 원장 행이 실제 Deposit/Bill/Room 값을 사용하고 내부 ID를 표시 필드에 포함하지 않음
- API: 출금 행이 확정·정정 비용만 포함하고 draft·void를 제외함
- API: 기본 기간이 실제 최초 기록 월에서 잘리고 기록이 없으면 빈 추세를 반환함
- 웹: 기본 최근 최대 6개월, 3/6/12개월, 직접 설정, 최근순·과거순 전환
- 웹: 입금 상세에서 `연결 청구`, 반복 `청구 구성`, ISO 시간 문자열을 노출하지 않음
- 웹: 입금 상세에서 청구월과 청구 내역을 분리하고 이번 입금·누적 수납·미수금을 서로 다른 의미로 표시함
- 웹: 월중 수납 흐름의 실제 월 범례, 좌우 막대 순서, 일별 눈금, 실제 말일, 지난달 기록 없음 안내
- 웹: 제목·범례와 하단 분석 영역 사이에 공통 1px 구분선을 사용함
- 웹: 6행 고정 영역과 내부 스크롤 스타일
- 웹: 내부 청구서 ID와 기존 임시 생성 함수·샘플 출금이 렌더 경로에서 제거됨
- 기존 청구수납 대상 테스트와 types, API, web 빌드
