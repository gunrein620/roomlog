# 관리자 일반 문의 미확인 배지 설계

## 목표

임차인이 보낸 일반 문의를 관리인이 아직 확인하지 않았다면 관리자 사이드바의 `소통·공지` 오른쪽에 미확인 메시지 총개수를 숫자 배지로 표시한다. 관리인이 해당 대화를 열면 그 스레드의 관리인 미확인 수를 0으로 처리한다.

## 범위

- `general` 맥락의 메시지 스레드만 사이드바 배지 집계에 포함한다.
- 임차인이 메시지를 보낼 때 해당 스레드의 관리인 미확인 수를 증가시킨다.
- 관리인이 스레드 상세를 열 때 해당 스레드의 관리인 미확인 수를 0으로 만든다.
- 관리인이 보낸 메시지는 관리인 미확인 수를 증가시키지 않는다.
- 미확인 총합이 0이면 배지를 렌더하지 않는다.
- 기존 `unreadCount`는 임차인이 아직 읽지 않은 관리인 발신 메시지 수이므로 의미와 동작을 변경하지 않는다.

## 데이터 계약

공유 `Thread` 타입과 API 저장 모델에 `managerUnreadCount`를 추가한다. 기본값은 0이다.

- 임차인 발신: `managerUnreadCount += 1`
- 관리인 읽음 처리 API: `managerUnreadCount = 0` 후 저장
- 관리인 목록 조회: 저장된 값을 그대로 반환
- 기존 데이터: DB 기본값 0으로 호환

읽음 처리는 조회 `GET`의 부수 효과로 만들지 않고 명시적인 `POST /manager/messaging/threads/:threadId/read`로 제공한다. 이 API도 기존 관리인 상세 조회와 같은 인증·호실 권한 검사를 수행한다.

## UI와 데이터 흐름

`ManagerAppShell`의 클라이언트 훅이 기존 Next 관리자 BFF(`/api/manager/...`)를 통해 일반 문의 목록을 조회하고 `managerUnreadCount` 합계를 계산한다. 계산값은 데스크톱·모바일 `ManagerSidebar`에 함께 전달한다. 경로가 바뀔 때 즉시 다시 조회하고 10초 간격으로 갱신해 새 문의와 상세 화면 진입으로 읽음 처리된 결과를 반영한다. `ManagerSidebar`는 `소통·공지` 상위 토글의 라벨과 펼침 아이콘 사이에 숫자 배지를 표시한다.

배지는 토큰 기반 스타일만 사용하고, 화면 판독기를 위해 `미확인 메시지 N개` 접근성 레이블을 제공한다. 큰 숫자는 레이아웃을 보호하기 위해 `99+`로 표시하되 접근성 레이블에는 실제 합계를 제공한다.

## 오류 처리

사이드바 집계 조회가 실패하면 기존 관리자 화면 진입을 막지 않고 배지만 숨긴다. 메시징 상세와 목록의 기존 API 오류 처리는 유지한다.

## 테스트

- API 도메인 테스트
  - 임차인 일반 문의 발신 시 관리인 미확인 수 증가
  - 관리인 발신 시 증가하지 않음
  - 관리인이 상세을 열면 0으로 초기화되고 목록에도 반영
  - 다른 관리인은 권한 없는 스레드를 읽음 처리할 수 없음
- web 테스트
  - 사이드바가 미확인 합계를 받아 `소통·공지` 오른쪽 숫자 배지를 렌더
  - 0이면 배지를 렌더하지 않음
  - 합계는 일반 문의만 포함
- 브라우저 확인
  - 미확인 일반 문의가 있는 계정에서 사이드바 배지 노출
  - 해당 대화를 연 뒤 배지가 사라지거나 감소

## 제외 범위

- 공지의 읽음·확인 상태 집계
- 하자·청구·계약·퇴실 맥락 스레드의 사이드바 알림
- 실시간 웹소켓 갱신과 브라우저 푸시 알림(사이드바 숫자는 10초 폴링으로 갱신)
- 기존 임차인용 `unreadCount` 의미 변경
