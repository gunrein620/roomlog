# 공지 번역 카드 축소·확장 설계

## 목표

`/manager/messaging/01`에서 번역하지 않은 언어 카드는 헤더만 보이는 작은 높이로 표시한다. 관리자가 번역 버튼을 누르면 해당 카드만 즉시 펼치고, 번역 결과와 검수 입력 영역을 제공한다.

## 범위

- English, 中文, Tiếng Việt 번역 카드
- 새 공지의 번역 전 상태
- 번역 요청 중, 성공, 실패 상태
- 저장된 번역이 있는 공지의 초기 상태

번역 API, 저장 형식, 다른 관리자 화면과 인프라 파일은 변경하지 않는다.

## 카드 상태

### 축소 상태

- 번역 제목과 본문이 모두 비어 있고 현재 번역 요청 중인 언어가 아니면 축소한다.
- 카드에는 언어 검수 라벨과 번역 버튼만 표시한다.
- 제목 입력란, 본문 입력란, 검수 완료 체크박스는 렌더링하지 않는다.

### 확장 상태

다음 조건 중 하나를 만족하면 해당 언어 카드를 확장한다.

- 해당 언어의 번역 버튼을 눌러 번역 요청이 진행 중이다.
- 번역 제목 또는 본문이 존재한다.
- 번역 요청이 실패했지만 사용자가 해당 카드를 이미 펼쳤다.

확장 상태에서는 제목 입력란, 본문 입력란, 검수 완료 체크박스를 표시한다.

## 상호작용

- 번역 버튼을 누르는 즉시 해당 카드만 확장한다.
- 요청 중에는 버튼 문구를 `번역 중...`으로 표시한다.
- 번역이 완료되면 기존 번역 API 응답을 제목과 본문에 채운다.
- 결과가 들어오면 검수 완료 체크박스를 활성화한다.
- 다른 언어 카드는 각 언어의 번역 버튼을 누르기 전까지 축소 상태를 유지한다.
- 번역 요청이 실패하면 카드는 확장 상태를 유지하고 기존 오류 메시지를 표시한다.
- 실패 후 같은 번역 버튼으로 다시 요청할 수 있다.

## 저장된 공지

- 저장된 제목이나 본문이 있는 언어 카드는 화면 진입 시 확장한다.
- 저장된 검수 상태와 입력값은 기존대로 유지한다.
- 번역 내용이 없는 언어만 축소한다.

## 구현

- `AnnouncementComposer`에서 사용자가 펼친 언어 목록을 로컬 상태로 관리한다.
- 언어별 `isExpanded`는 사용자 확장 상태, 번역 진행 상태, 번역 제목·본문 존재 여부로 계산한다.
- 번역 버튼 처리 시작 시 해당 언어를 사용자 확장 상태에 추가한다.
- `isExpanded`가 참일 때만 입력 영역과 검수 행을 렌더링한다.
- 축소 카드는 별도 고정 높이를 만들지 않고 기존 패딩과 헤더 높이로 자연스럽게 결정한다.
- 새 색상값을 추가하지 않고 기존 디자인 토큰만 사용한다.

## 접근성

- 번역 버튼에 `aria-expanded`를 제공해 현재 카드 상태를 알린다.
- 입력 영역에는 안정적인 언어별 `id`를 제공하고 번역 버튼의 `aria-controls`와 연결한다.
- 축소 상태의 입력 영역은 시각적으로만 숨기지 않고 DOM에서 제거한다.

## 테스트

- 번역 내용이 없는 카드가 축소 상태인지 확인한다.
- 번역 버튼이 `aria-expanded=false`에서 클릭 직후 `true`로 바뀌는지 확인한다.
- English 버튼을 눌렀을 때 English 입력 영역만 나타나는지 확인한다.
- 번역 완료 후 English 결과와 활성화된 검수 체크박스를 확인한다.
- 中文, Tiếng Việt 카드는 계속 축소 상태인지 확인한다.
- 저장된 번역이 있는 카드는 처음부터 펼쳐지는 조건을 단위 테스트로 고정한다.
- web 단위 테스트, Next.js 빌드, Docker 브라우저 검증을 실행한다.
