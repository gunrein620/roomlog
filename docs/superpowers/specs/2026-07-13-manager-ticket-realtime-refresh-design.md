# 민원/하자 관리 실시간 목록 갱신 설계

## 목표

세입자가 민원 또는 하자를 등록하면 이미 열려 있는 관리자 `민원/하자 관리` 화면에 새 항목이 새로고침 없이 표시되도록 한다.

## 현재 문제와 원인

- 관리자 목록 페이지는 서버 컴포넌트가 조회한 `rows`를 클라이언트 컴포넌트에 최초 한 번 전달한다.
- 세입자 민원 등록 후 관리자 페이지에 서버 데이터를 다시 요청하는 동작이 없다.
- `POST /tenant/complaints`와 전화 접수 API는 민원 생성 성공 후 실시간 활동 이벤트를 발행하지 않는다.
- 따라서 DB에는 접수 내용이 저장되지만 관리자 목록은 브라우저 새로고침 전까지 이전 스냅샷을 유지한다.

## 설계

### API 이벤트 발행

세입자 민원 생성이 성공한 뒤 기존 `RealtimeGateway`를 통해 다음 활동 이벤트를 발행한다.

```text
event: roomlog:activity
payload: { kind: "ticket" }
```

일반 폼 접수와 전화 접수 양쪽에 동일한 이벤트를 적용한다. 도메인 서비스가 성공적으로 민원과 티켓을 생성한 뒤에만 이벤트를 보내며, 검증 실패나 예외가 발생한 경우에는 보내지 않는다.

### 관리자 화면 갱신

관리자 민원/하자 목록 전용 클라이언트 컴포넌트를 추가한다.

- WebSocket이 연결된 상태에서 `roomlog:activity`의 `kind`가 `ticket`이면 즉시 `router.refresh()`를 호출한다.
- WebSocket이 끊긴 상태에서는 3초 간격 폴링으로 갱신한다.
- WebSocket 연결 중에도 30초 안전 갱신을 유지해 이벤트 유실 후 장시간 오래된 목록이 남지 않게 한다.
- 창이 다시 포커스를 얻거나 문서가 다시 보이는 상태가 되면 즉시 갱신한다.
- 문서가 백그라운드 상태이거나 input, textarea, select, contenteditable 요소를 조작 중이면 갱신을 미뤄 필터 조작을 방해하지 않는다.
- 컴포넌트는 관리 화면에만 마운트하고 민원 대시보드에는 적용하지 않는다.

`router.refresh()`는 현재 URL의 `view`, `type` 쿼리와 클라이언트 컴포넌트의 로컬 필터 상태를 유지하면서 서버 컴포넌트 데이터만 다시 요청한다. 갱신된 `rows`가 전달되면 상태별 집계, 건물·담당자 옵션, 표와 페이지네이션이 함께 다시 계산된다.

## 오류 처리

- 소켓 연결 실패는 사용자 오류로 표시하지 않고 3초 폴링으로 자동 전환한다.
- 일시적인 서버 조회 실패는 기존 API 클라이언트 정책에 따라 빈 결과를 반환할 수 있으므로, 안전 갱신과 다음 이벤트에서 다시 동기화한다.
- 소켓 이벤트 자체에는 민원 내용이나 사용자 정보를 싣지 않고 변경 신호만 전달한다. 실제 데이터와 권한 검증은 기존 관리자 REST 조회가 담당한다.

## 테스트

### API

- 일반 민원 생성 성공 후 `roomlog:activity`와 `{ kind: "ticket" }`가 정확히 한 번 발행되는지 검증한다.
- 전화 민원 생성 성공에도 같은 이벤트가 발행되는지 검증한다.

### Web

- 전용 자동 갱신 컴포넌트가 `ticket` 활동 이벤트를 구독하고 `router.refresh()`를 호출하는 계약을 검증한다.
- 소켓 연결 상태에 따른 3초 폴백과 30초 안전 갱신, focus/visibility 재동기화 및 cleanup을 검증한다.
- 관리 화면 페이지에만 자동 갱신 컴포넌트가 마운트되는지 검증한다.
- 기존 민원/하자 필터, 표, 페이지네이션 테스트가 계속 통과하는지 확인한다.

## 범위 제외

- Socket.IO 인프라나 Docker, 배포 환경 설정 변경
- 서버가 목록 전체를 push하는 방식
- 사용자별 WebSocket room 세분화
- 화면 내 별도 토스트 또는 신규 항목 강조 애니메이션
- 민원 대시보드의 실시간 갱신

## 완료 기준

- 관리 화면을 열어 둔 상태에서 다른 세입자 세션으로 민원/하자를 등록하면 새 행이 즉시 표시된다.
- WebSocket을 사용할 수 없는 상태에서도 최대 3초 안에 표시된다.
- 선택한 유형, 상태, 담당자, 건물 필터와 현재 관리 화면 URL이 갱신 후 유지된다.
- API, Web 대상 테스트와 기본 검증이 통과하며 인프라 파일 변경이 없다.
