# 관리자 공지 대상 범위 Pill 버튼 설계

## 목표

`/manager/messaging/01`의 `전체·건물·호실` 대상 범위를 `긴급·생활·행사`와 동일한 클릭형 pill 버튼으로 표시한다.

## 범위

- 수정 화면: `/manager/messaging/01`
- 수정 대상: 공지 작성 컴포저 스타일과 관련 web 계약 테스트
- 제외 대상: 대상 범위 상태 계산, 건물·호실 선택 흐름, API, `/manager/messaging/00`, `/02`, 인프라 설정

## UI 동작

- `전체`, `건물`, `호실`은 한 행의 pill 버튼으로 표시한다.
- 선택된 범위는 기본 색상 배경과 반전된 글자색을 사용한다.
- 선택되지 않은 범위는 낮은 강조의 배경과 보조 글자색을 사용한다.
- 기존 원형 라디오 마크는 제거한다.
- pill의 높이, 안쪽 여백, 모서리, 글자 굵기와 상태 전환은 카테고리 pill 규칙을 재사용한다.

## 접근성

- 실제 `input type="radio"`와 `name="scope"`는 유지한다.
- 각 pill은 연결된 label 내부에서 클릭되므로 마우스와 터치로 선택할 수 있다.
- 키보드 포커스 시 기존 focus ring을 유지한다.
- 선택 상태는 CSS의 `input:checked + pill` 규칙으로 표현한다.

## 기존 기능 보존

- `changeScope` 호출과 호실 범위 진입 시 기존 대상 초기화 동작을 유지한다.
- `건물` 선택 시 건물 드롭다운을 표시한다.
- `호실` 선택 시 건물 선택 후 해당 건물의 호실 목록을 표시한다.
- 대상 요약 및 저장 payload는 변경하지 않는다.

## 테스트

- 소스 계약 테스트에서 범위 선택이 카테고리와 같은 pill 클래스를 사용하는지 확인한다.
- 원형 라디오 마크 렌더링이 제거됐는지 확인한다.
- 범위 라디오 input과 `changeScope` 연결이 유지되는지 확인한다.
- web 단위 테스트와 프로덕션 빌드를 실행한다.
- Docker 브라우저에서 선택 전환 시 파란 활성 pill이 하나만 표시되고 건물·호실 컨트롤이 기존대로 전환되는지 확인한다.

## 완료 조건

- `전체·건물·호실`이 `긴급·생활·행사`와 같은 pill 버튼 모양으로 표시된다.
- 선택 상태, 키보드 포커스와 기존 대상 선택 기능이 유지된다.
- `/manager/messaging/01` 외 제품 화면과 인프라 파일은 변경되지 않는다.
