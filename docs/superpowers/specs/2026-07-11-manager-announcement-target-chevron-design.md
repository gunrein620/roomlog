# 공지 대상 결과 박스 화살표 제거 설계

## 범위

- 대상 화면은 `/manager/messaging/01`로 한정한다.
- 건물 선택 컨트롤의 드롭다운 화살표는 유지한다.
- 클릭 동작이 없는 대상 결과 박스에서는 `V/⌄` 화살표를 완전히 제거한다.
- 다른 관리자 화면, API 계약, 인프라 파일은 수정하지 않는다.

## UI 설계

### 화살표 처리

- 위 건물 `<select>`는 실제 선택 컨트롤이므로 브라우저 기본 드롭다운 화살표를 유지한다.
- 아래 대상 결과 박스는 조회 전용이므로 화살표와 클릭 커서를 표시하지 않는다.
- 문자형 `V`, `⌄` 및 장식용 화살표 아이콘을 대상 결과 박스에 넣지 않는다.

### 대상 결과 표시

- 대상 결과 박스는 기존처럼 현재 타깃 라벨만 표시한다.
- 전체·건물·호실 범위가 바뀌면 기존 `buildAnnouncementTarget` 결과에 따라 라벨이 즉시 갱신된다.
- 대상 변경은 기존 건물 선택 및 호실 체크 컨트롤에서만 수행한다.

## 기능 영향

- 기존 전체·건물·호실 대상 계산 방식은 변경하지 않는다.
- 번역, 검수 무효화, 임시 저장, 검토 화면 이동, 긴급 공지 게이트는 변경하지 않는다.
- 대상 결과 박스는 비대화형 표시 요소로 유지한다.

## 테스트 및 검증

- 소스 계약 테스트에서 문자형 `⌄`가 제거됐는지 확인한다.
- 대상 결과 박스에 별도 화살표 요소가 없는지 확인한다.
- 대상 결과 박스가 버튼·`details`가 아닌 비대화형 표시 요소인지 확인한다.
- 공지 작성 상태 유닛 테스트와 기존 공지 작성 소스 테스트를 실행한다.
- 웹 유닛 테스트, Next 빌드, Docker 브라우저에서 다음을 확인한다.
  - 위 건물 선택 화살표만 표시
  - 아래 대상 결과 박스 화살표 미표시
  - 범위 변경 시 라벨과 목록 갱신
  - 번역 버튼과 임시 저장 정상 동작
  - 브라우저 콘솔 오류 및 Next 오류 오버레이 없음
