# Manager Messaging Reply Badge Design

## Goal

관리자 메시징 목록 카드의 `답장 필요` 배지가 좁은 너비에서 `답장 필 / 요`로 불균형하게 자동 줄바꿈되는 문제를 해결한다.

## Scope

- `/manager/messaging/00` 티켓 카드의 답장 필요 배지만 수정한다.
- 상단 집계 문구 `답장 필요 N건`과 섹션 제목은 변경하지 않는다.
- 공용 `Badge` 컴포넌트와 다른 화면의 배지는 변경하지 않는다.
- 인프라 파일은 수정하지 않는다.

## Presentation

배지의 시각 텍스트를 다음 두 줄로 고정한다.

```text
답장
필요
```

배지 내부에 전용 span을 사용해 세로 배치하고, 각 단어는 `white-space: nowrap`으로 글자 단위 분리를 막는다. 두 줄은 수평·수직 가운데 정렬하며 현재 배지 색상과 테두리는 유지한다.

스크린리더에는 중복되거나 끊어진 텍스트 대신 `답장 필요` 한 문구로 전달되도록 전용 span에 `aria-label="답장 필요"`를 지정한다.

## Testing

1. 목록 페이지 소스 계약 테스트에 전용 두 줄 구조와 접근성 라벨을 먼저 추가해 실패를 확인한다.
2. 최소 JSX와 토큰 기반 inline style을 구현한다.
3. 집중 테스트, 전체 web 단위 테스트, `scripts/verify.sh`를 실행한다.
4. Docker Desktop은 중복 실행하지 않고 현재 데몬이 실행 중일 때 web 이미지만 재빌드한다.
5. 로컬 페이지를 열어 변경 결과를 확인한다.

## Success Criteria

- 모든 답장 필요 배지가 정확히 `답장` / `필요` 두 줄로 표시된다.
- `필`과 `요`가 서로 다른 줄로 갈라지지 않는다.
- 스크린리더는 `답장 필요`로 읽는다.
- 다른 배지와 메시징 기능의 동작은 변하지 않는다.
