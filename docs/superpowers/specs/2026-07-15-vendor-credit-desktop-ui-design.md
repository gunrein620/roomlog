# 업체관리 크레딧·결제 데스크톱 UI 설계

## 목표

관리자용 `크레딧·결제` 화면을 모바일 카드 나열처럼 보이는 현재 구성에서 데스크톱 운영 콘솔 밀도로 정리한다. 데이터 계약, 결제 정책, 지급·취소 조건과 API 호출은 변경하지 않는다.

## 범위

- 상단 크레딧 요약의 높이와 여백을 줄이고 하나의 요약 스트립처럼 보이게 한다.
- 자동결제 정책 선택, 한도 입력, 저장 액션의 시각적 위계와 배치를 개선한다.
- 업체 지급 요청의 정보·상태·액션을 데스크톱 열 구조로 정돈한다.
- 크레딧 원장과 충전 내역의 표·행 밀도를 높인다.
- 크레딧 원장과 충전 내역이 각각 8개 항목을 초과해 세로로 길어질 때 해당 영역 내부에서 스크롤되게 한다.
- 기존 토큰만 사용하고 raw hex 색상은 추가하지 않는다.

## 비범위

- 크레딧 잔액, 자동결제 정책, 지급 요청의 데이터 모델 변경
- API 또는 서버 액션 변경
- 충전 모달과 토스 결제 흐름 변경
- 다른 업체관리 탭의 UI 개편
- 모바일 전용 화면 설계

## 레이아웃

### 상단 요약

현재 3개의 독립된 큰 카드 대신 데스크톱 요약 스트립을 사용한다. `현재 크레딧`, `지급 처리 필요`, `현재 자동결제 기준`을 동일한 높이의 세 칸으로 배치하고 칸 사이는 경계선으로 구분한다. 현재 크레딧만 주 강조색을 사용하며, 충전 버튼은 이 칸의 우측에 둔다. 카드 최소 높이와 내부 패딩은 다른 관리자 청구 화면의 요약 스트립 수준으로 줄인다.

### 자동결제 정책

정책 패널은 다음 세 구역으로 나눈다.

1. 제목과 설명
2. 두 개의 가로형 선택 카드
3. 설정 및 저장 액션 바

라디오 기본 원형 컨트롤은 시각적으로 숨기되 키보드와 스크린 리더 입력은 유지한다. 각 선택 카드는 사용자에게 큰 라디오 옵션으로 동작하며 선택 상태에서는 테두리, 배경, 선택 표시 아이콘이 함께 바뀐다.

선택 표시 원은 카드 높이를 지배하지 않도록 `calc(var(--space-lg) + var(--space-xs))`(20px) 크기로 제한한다. 카드 전체와 연결된 실제 라디오 레이블의 클릭 영역은 기존 크기를 유지해 마우스와 키보드 조작성을 줄이지 않는다.

`한도 이하 자동 차감`을 선택하면 한도 입력을 선택 카드 내부나 저장 버튼 옆에 끼우지 않고, 선택지 아래의 독립된 설정 행에 표시한다. `정책 저장` 버튼은 항상 패널 하단 우측의 액션 바에 유지한다. 따라서 한도 입력의 표시 여부와 관계없이 버튼 위치가 움직이지 않는다.

### 업체 지급 요청

각 요청은 데스크톱에서 `업체·작업`, `금액·상태`, `처리 컨트롤`의 열 구조로 정렬한다. 메타 정보와 입력은 작은 캡션 밀도를 사용하며 버튼은 내용 폭만 차지한다. 직접 결제 입력은 한 행에 배치하고, 취소·정정 사유와 위험 액션은 별도 보정 영역으로 구분한다. 기존 상태별 액션 노출과 확인 대화상자는 그대로 유지한다.

### 원장과 충전 내역

데스크톱에서는 크레딧 원장을 더 넓게, 충전 내역을 더 좁게 둔 2열 구성을 유지한다. 두 패널의 헤더와 본문 정렬을 맞추고 표 셀 및 충전 행의 패딩을 줄인다.

각 내역 영역은 최대 8개 항목이 한 번에 보이는 높이를 기준으로 제한한다. 항목이 더 많으면 그 영역만 세로 스크롤되고 페이지 전체가 계속 길어지지 않는다. 크레딧 원장의 표 헤더는 내부 스크롤 중 상단에 고정한다. 스크롤 컨테이너에는 접근 가능한 레이블과 키보드 포커스를 제공한다. 서버 페이지네이션의 `이전 내역 더 보기` 버튼은 스크롤 영역의 마지막 항목으로 유지한다.

## 반응형 처리

화면은 데스크톱 우선으로 설계한다. 관리자 셸의 콘텐츠 폭을 그대로 사용하며 별도의 모바일 프레임이나 좁은 최대 폭을 추가하지 않는다. 1180px 이하에서는 요약 및 이력 2열을 단계적으로 줄이고, 900px 이하에서는 정책 카드와 지급 요청을 1열로 전환해 레이아웃 파손만 방지한다.

## 상태와 접근성

- 데모 읽기 전용, 저장 중, 성공·오류 피드백 동작은 유지한다.
- 정책 선택은 실제 `input[type="radio"]`와 `label` 관계를 유지한다.
- `:focus-visible` 상태는 기존 포커스 토큰을 사용한다.
- 선택 상태를 색상만으로 알리지 않고 선택 표시 아이콘과 텍스트를 함께 사용한다.
- 내부 스크롤 영역은 마우스 휠, 트랙패드, 키보드로 탐색 가능해야 한다.
- 색상, 간격, 반경, 그림자, 폰트 크기는 `packages/ui/src/tokens.css`의 변수만 사용한다.

## 구현 경계

주요 변경 파일은 다음 두 파일로 제한한다.

- `apps/web/src/app/manager/vendor-mgmt/credit/CreditWorkspace.tsx`
- `apps/web/src/app/manager/vendor-mgmt/credit/CreditWorkspace.module.css`

필요한 UI 계약 테스트는 기존 `credit-workspace.spec.ts`에 추가한다. 공유 타입과 API는 수정하지 않는다.

## 검증

- 정책 선택 카드가 실제 라디오 입력과 연결돼 있는지 확인한다.
- 정책 선택 표시 원이 20px 토큰 크기이고 카드 클릭 영역은 유지되는지 확인한다.
- 정책 저장 버튼이 별도의 하단 액션 영역에 존재하는지 확인한다.
- 한도 입력 토글 전후에도 저장 액션 영역 구조가 유지되는지 확인한다.
- 원장과 충전 내역에 각각 전용 스크롤 컨테이너가 있고 원장 헤더가 고정되는지 확인한다.
- 데모 읽기 전용과 기존 지급·취소 액션 테스트가 계속 통과하는지 확인한다.
- web 단위 테스트와 web 빌드를 실행한다.
- docker-compose web 이미지를 재빌드한 뒤 데스크톱 브라우저에서 정책, 지급 요청, 내역 스크롤을 시각 검수한다.
