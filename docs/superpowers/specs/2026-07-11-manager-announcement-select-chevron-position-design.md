# 공지 건물 선택 화살표 위치 조정 설계

## 범위

- 대상 화면은 `/manager/messaging/01`로 한정한다.
- 위 건물 선택 `<select>`의 화살표를 현재보다 약 10px 왼쪽으로 이동한다.
- 아래 대상 결과 박스는 화살표가 없는 현재 상태를 유지한다.
- 대상 계산, 번역, 저장, 검토 이동, 다른 화면 및 인프라는 변경하지 않는다.

## 구현 방식

- 브라우저 기본 `<select>` 화살표는 위치를 조정할 수 없으므로 `appearance: none`으로 숨긴다.
- `<select>`를 `.selectWrap` 요소로 감싸고 `::after` 가상 요소로 얇은 화살표를 그린다.
- 화살표 오른쪽 위치는 기존 기본 여백으로 간주하는 `var(--space-lg)`에 `10px`을 더한 `calc(var(--space-lg) + 10px)`로 고정한다.
- 화살표는 CSS 테두리로 그리고 색상은 `var(--on-surface)` 토큰을 사용한다.
- 가상 요소는 `pointer-events: none`으로 설정해 `<select>` 클릭과 키보드 동작을 방해하지 않는다.
- `<select>` 오른쪽 패딩을 늘려 긴 건물명이 화살표와 겹치지 않게 한다.

## 접근성 및 동작

- 실제 폼 컨트롤은 기존 네이티브 `<select aria-label="공지 대상 건물">`를 그대로 사용한다.
- 마우스, 키보드, 보조기기의 선택 동작은 변경하지 않는다.
- 화살표는 장식 요소이므로 접근성 트리에 추가하지 않는다.

## 테스트 및 검증

- 소스 계약 테스트에서 `.selectWrap`이 `<select>`를 감싸는지 확인한다.
- CSS 계약에서 `appearance: none`, `right: calc(var(--space-lg) + 10px)`, `pointer-events: none`을 확인한다.
- 공지 작성 소스 테스트와 웹 유닛 테스트를 실행한다.
- Next 프로덕션 빌드를 실행한다.
- Docker 브라우저에서 화살표 위치, 건물 선택 동작, 대상 라벨 갱신, 번역·저장 버튼 활성, 콘솔 오류 유무를 확인한다.
