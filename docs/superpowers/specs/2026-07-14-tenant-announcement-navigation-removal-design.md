# 세입자 공지 목록 내비게이션 제거 설계

## 목표

`/tenant/messaging/02` 공지 목록에서 데스크톱 상단 및 모바일 하단에 표시되는 `홈 · 공지 · 고정비 · 내 정보` 주요 메뉴를 제거한다. 공지 제목, 검색, 필터, 목록, 문의 영역은 그대로 유지한다.

## 변경 범위

- `AnnouncementListPage.tsx`에서 전용 주요 메뉴 데이터와 내비게이션 마크업을 제거한다.
- 더 이상 사용하지 않는 내비게이션 아이콘 타입과 아이콘 import를 정리한다.
- `AnnouncementListPage.module.css`에서 `bottomNav`, `navLink`, `navLinkActive` 규칙과 관련 포커스 선택자를 제거한다.
- 모바일 콘텐츠의 하단 내비게이션 예약 여백을 일반 페이지 여백으로 줄인다.
- 다른 세입자 화면의 공용 셸이나 내비게이션은 변경하지 않는다.

## 반응형 동작

- 모바일: 화면 하단 고정 메뉴가 사라지고 공지 콘텐츠가 자연스럽게 페이지 끝까지 이어진다.
- 태블릿·데스크톱: 헤더 위에 있던 가로 주요 메뉴가 사라지고 공지 헤더가 카드의 첫 영역이 된다.
- 기존 검색, 필터, 공지 카드 그리드 반응형 규칙은 유지한다.

## 테스트와 검증

1. 계약 테스트에 주요 메뉴 마크업과 관련 CSS 규칙이 없음을 명시하고 먼저 실패를 확인한다.
2. 최소 코드 변경 후 해당 계약 테스트와 web 전체 테스트를 통과시킨다.
3. Docker web 이미지를 재빌드하고 공지 목록을 데스크톱·모바일 뷰포트에서 확인한다.
4. 공지 제목, 검색, 필터, 목록이 유지되고 런타임 오류가 없는지 확인한다.

## 비범위

- 다른 세입자 페이지 내비게이션 변경
- 공지 상세 화면 변경
- API, 데이터 모델, 인증, Docker 및 배포 설정 변경
