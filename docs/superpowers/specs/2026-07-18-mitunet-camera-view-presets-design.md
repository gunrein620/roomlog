# MitUNet 카메라 뷰 프리셋 설계

## 목표

도면의 3D 변환이 끝난 뒤 사용자가 구조 확인과 가구 배치 중에도 공간을 여러 고정 각도에서 빠르게 확인할 수 있게 한다. 모든 고정 뷰 전환은 부드러운 카메라 애니메이션을 사용하고, 자동 둘러보기는 사용자의 직접 조작을 방해하지 않아야 한다.

## 적용 범위

- MitUNet 변환 화면의 3D 상태와 가구 배치 상태에 적용한다.
- 원본 도면 편집 상태에서는 뷰 프리셋을 숨긴다.
- 저장된 RoomLog 매물 3D 화면은 이번 범위에 포함하지 않는다.
- 기존 OrbitControls의 회전, 확대, 이동 동작은 유지한다.

## 뷰 구성

뷰 컨트롤은 `입체`, `위`, `정면`, `왼쪽`, `오른쪽`, `자동 둘러보기` 여섯 개로 구성한다.

- 입체: 현재 기본 대각선 원근 뷰로 돌아간다.
- 위: 공간 중심을 수직으로 내려다본다.
- 정면: 도면 앞쪽에서 전체 구조를 바라본다.
- 왼쪽: 도면 왼쪽에서 전체 구조를 바라본다.
- 오른쪽: 도면 오른쪽에서 전체 구조를 바라본다.
- 자동 둘러보기: 현재 공간 중심과 거리를 유지하면서 수평 방향으로 천천히 회전한다.

고정 뷰의 카메라 거리는 현재 도면의 바운딩 반경과 화면 비율에서 계산하므로 도면 크기에 관계없이 전체 모델이 프레임 안에 들어와야 한다.

## UI와 상태

- 기존 원본/3D 전환 아래에 별도의 뷰 프리셋 행을 둔다.
- 3D 모델이 준비된 경우에만 활성화한다.
- 가구 배치 패널이 열려 있어도 프리셋 행을 유지한다.
- 작은 화면에서는 프리셋 행을 가로 스크롤로 제공한다.
- 선택된 고정 뷰와 실행 중인 자동 둘러보기는 활성 스타일로 표시한다.
- 사용자가 OrbitControls로 카메라를 직접 조작하면 고정 뷰 활성 표시는 해제한다.

## 카메라 전환

- 고정 뷰 버튼은 카메라 위치와 OrbitControls 대상점을 약 700ms 동안 cubic-out 보간한다.
- `prefers-reduced-motion`이 활성화된 경우 애니메이션 없이 즉시 이동한다.
- 전환 중에는 카메라 입력만 잠시 막고 가구의 선택 및 배치 상태는 변경하지 않는다.
- 진행 중인 전환에서 다른 프리셋을 누르면 기존 전환을 취소하고 새 목표로 이동한다.

## 자동 둘러보기

- 자동 둘러보기 버튼은 시작과 정지를 전환하는 토글이다.
- 회전은 OrbitControls 대상점을 중심으로 수평 궤도를 따라 일정한 저속으로 진행한다.
- 다른 프리셋 선택, 사용자의 드래그 시작, 원본 도면 전환 시 즉시 정지한다.
- 가구를 선택하거나 배치 모드에 진입하는 것만으로는 자동 회전을 강제 종료하지 않는다.
- 자동 회전 중에도 확대·축소는 허용하며 현재 반경을 이후 회전에 사용한다.

## 구현 경계

- 카메라 목표 위치 계산은 순수 함수로 분리해 각 뷰의 방향과 프레이밍을 단위 테스트한다.
- UI 이벤트는 뷰 이름만 카메라 컨트롤러에 전달한다.
- 기존 원본/3D 전환이 사용하는 카메라 보간 루프를 확장해 중복 애니메이션 루프를 만들지 않는다.
- 가구 데이터, 벽 좌표, 바닥재 좌표는 변경하지 않는다.

## 검증

- 각 프리셋이 서로 다른 유효한 카메라 위치를 계산하는지 테스트한다.
- 전환 중 새 프리셋을 선택했을 때 최종 목표가 교체되는지 테스트한다.
- 자동 둘러보기가 토글되고 사용자 조작과 원본 전환에서 정지하는지 테스트한다.
- 가구 배치 상태에서 버튼이 유지되고 선택 및 임시 배치 상태가 보존되는지 정적 회귀 테스트로 고정한다.
- reduced-motion 환경에서 즉시 이동하는지 테스트한다.
