# 사는집 집주인 공지 카드 전체 링크 설계

## 목표

`/living`의 실제 집주인 공지 카드에서 아이콘, 제목, 본문, 날짜, 여백을 포함한 카드 전체를 클릭하면 `/tenant/messaging/02` 공지 목록으로 이동하게 한다.

## 변경 범위

- `TenantMyPage.tsx`의 공지 준비 상태에서 카드 콘텐츠 전체를 `Link href="/tenant/messaging/02"` 내부에 배치한다.
- 기존 공지 상세 URL 생성기 `tenantAnnouncementDetailHref` import와 사용을 이 화면에서 제거한다.
- 전용 `tenant-announcement-link` 클래스로 링크가 카드 전체를 채우고 기존 반응형 그리드와 여백을 유지하게 한다.
- 로딩, 빈 상태, 오류 상태 카드는 현재처럼 링크 없이 상태 문구만 보여 준다.

## 구조와 반응형

- `tenant-announcement-card`는 카드 외곽선, 배경, 최소 높이, 오버플로를 담당한다.
- `tenant-announcement-link`는 카드 내부 전체 영역을 채우며 아이콘과 텍스트의 2열 그리드를 담당한다.
- 데스크톱은 기존 `30px 32px`, 기본 화면은 `24px`, 560px 이하 모바일은 기존 `18px` 여백과 `54px` 아이콘 열을 유지한다.
- 링크의 글자색과 밑줄은 기존 카드 디자인을 유지하고, 키보드 포커스는 공유 색상 변수로 표시한다.

## 테스트와 검증

1. `property-shell.spec.mjs`에서 사는집 공지 카드가 목록 URL을 사용하고 상세 URL 생성기를 사용하지 않는다는 실패 계약을 먼저 추가한다.
2. 카드 전체 링크 구조와 CSS 계약을 추가하고 집중 테스트의 RED를 확인한다.
3. 최소 구현 후 집중 계약 테스트와 web 검증을 실행한다.
4. Docker web 이미지를 재빌드하고 카드 아이콘·여백 영역을 실제 클릭해 `/tenant/messaging/02` 도착과 오류 부재를 확인한다.

## 비범위

- 공지 목록 안의 개별 공지 상세 링크 변경
- 메시징 화면과 공지 상세 화면의 동선 변경
- 공지 데이터 조회, 로딩, 빈 상태, 오류 처리 변경
- 인프라 설정 변경
