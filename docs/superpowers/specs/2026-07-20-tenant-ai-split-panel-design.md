# 세입자 AI 비서 스플릿 패널 설계

## 목표

세입자 AI 비서의 진입과 대화 UX를 관리인 AI 비서와 맞춘다. 사용자는 별도의 텍스트·음성 선택 화면을 거치지 않고 텍스트 대화로 바로 진입하며, 같은 패널 안에서 음성으로 전환할 수 있다. 대화와 작성 중 상태는 현재 브라우저 탭이 유지되는 동안 보존한다.

## 화면 동작

### 데스크톱

- AI 비서 버튼을 누르면 기존 세입자 화면 오른쪽에 AI 사이드패널을 연다.
- 본문과 AI 패널은 관리인 화면처럼 좌우 스플릿뷰로 배치한다.
- 패널은 텍스트 모드로 바로 열리고, 하단 토글로 음성 모드를 전환한다.
- 패널을 닫아도 대화와 입력 초안은 지우지 않는다.

### 태블릿·모바일

- AI 패널이 열리면 기존 화면을 함께 좁히지 않고 AI 패널만 전체 화면에 표시한다.
- 닫기 버튼으로 기존 세입자 화면에 돌아간다.
- 텍스트·음성 전환과 대화 내용은 데스크톱과 동일하다.

## 상태 보존

- 관리인 AI와 같은 모듈 스코프 외부 스토어와 `useSyncExternalStore` 패턴을 사용한다.
- 메시지, 패널 열림 여부, 텍스트·음성 모드, 입력 중인 문장, 민원 초안 전환 상태를 저장한다.
- `sessionStorage`에 미러링하여 세입자 탭 이동, 컴포넌트 재마운트, 새로고침 후에도 복원한다.
- 진행 중 네트워크 요청을 의미하는 `busy` 상태는 복원하지 않는다.
- 브라우저 탭을 닫으면 세션은 자연스럽게 초기화된다.
- 민원 접수 완료 후에는 완료 영수증을 남기되, 다음 민원 상담을 위해 백엔드 intake 세션만 새로 시작한다. 기존 화면 대화는 사용자가 현재 탭을 닫기 전까지 유지한다.

## 민원 초안 전환

- 사용자가 민원 작성을 요청하면 AI 대화에 “접수 초안을 작성하겠습니다” 진행 메시지를 먼저 표시한다.
- 초안이 준비돼도 AI 패널을 강제로 닫지 않는다.
- 데스크톱에서는 오른쪽 AI 대화를 유지한 채 본문에 신규 요청 폼을 열고, AI가 채운 제목·내용·긴급도·방문 가능 시간을 보여준다.
- 태블릿·모바일에서는 AI 대화에서 신규 요청 폼 전체 화면으로 전환한다.
- 신규 요청 폼에는 “AI가 작성한 초안”이라는 출처 안내와 “AI 대화로 돌아가기” 동선을 제공한다.
- 사용자가 폼에서 사진을 추가하거나 내용을 수정한 뒤 기존 민원 접수 API로 제출한다.
- 폼을 닫거나 AI로 돌아가도 대화와 폼 입력 상태를 잃지 않는다.

## 컴포넌트 경계

- `TenantMyPage`는 세입자 본문과 AI 패널의 레이아웃 조합만 담당한다.
- 새 세입자 AI 스토어가 패널 열림, 모드, 메시지, 입력 초안과 초안 전환 상태를 담당한다.
- `useTenantAiAssistant`는 intake API와 Realtime 음성 연결을 담당하며, 화면 재마운트와 무관하게 스토어에 결과를 기록한다.
- AI 패널 UI는 관리인 패널의 구조와 공용 `manager-ai-*` 스타일을 재사용하되 세입자 데이터와 액션만 연결한다.

## 오류와 안전

- 마이크 연결 종료 또는 패널 닫기 시 오디오 트랙을 즉시 비활성화한다.
- 저장된 세션 데이터가 깨졌으면 안전하게 초기 상태로 시작한다.
- 네트워크 오류는 대화 안에 시스템 메시지로 표시하고 기존 대화는 유지한다.
- 민원 접수는 기존 서버 승인·접수 규칙을 그대로 사용한다.

## 검증

- 최초 진입에 모드 선택 화면이 없고 텍스트 대화가 바로 보이는지 확인한다.
- 데스크톱 스플릿뷰와 모바일 전체 화면 패널을 각각 검증한다.
- 닫기·재열기, 탭 이동, 새로고침 후 대화와 입력 초안이 복원되는지 검증한다.
- 텍스트와 음성 모드 전환 시 하나의 intake 세션과 대화를 공유하는지 검증한다.
- AI 초안 생성 후 패널이 유지되고 요청 폼이 채워지는지 검증한다.
- 요청 폼과 AI 대화를 왕복해도 사진과 수정 내용이 유지되는지 검증한다.
