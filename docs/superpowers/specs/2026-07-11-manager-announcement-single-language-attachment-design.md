# 공지 단일 언어 번역 첨부 설계

## 목표

관리자가 `/manager/messaging/01`에서 English, 中文, Tiếng Việt 중 한 언어만 번역해도 그 결과를 최종 공지 제목과 본문으로 첨부하고 기존 발송 흐름으로 전송할 수 있게 한다.

## 수정 범위

- `/manager/messaging/01` 공지 작성 화면과 화면 전용 로직
- `/manager/messaging/02` 발송 전 검토 화면
- 위 두 화면의 회귀 테스트

`/manager/messaging/00`, API, 공유 타입, 임차인 화면, 인프라 파일은 수정하지 않는다.

## 공지 작성 동작

### 번역

- 새 공지는 기존처럼 번역 카드가 축소된 상태로 시작한다.
- 언어별 번역 버튼을 눌렀을 때만 해당 언어를 번역한다.
- 한 언어의 제목과 본문만 완성돼도 그 카드에 `첨부하기` 버튼을 표시한다.
- 검수 완료 체크박스는 렌더링하지 않는다.

### 첨부

- `첨부하기`를 누르면 선택한 번역 제목과 본문으로 왼쪽 `공지 제목`, `상세 내용`을 교체한다.
- 선택된 카드의 버튼은 `첨부됨`으로 표시한다.
- 다른 언어를 첨부하면 최종 언어, 제목, 본문을 새 번역으로 교체한다.
- 첨부된 제목이나 본문을 사용자가 직접 수정하면 첨부 상태를 해제한다.
- 첨부 상태가 없으면 `검토하고 발송으로` 진행할 수 없다.

## 기존 API 호환 투영

기존 서버는 긴급 공지에 `en`, `zh`, `vi` 번역이 각각 정확히 하나 있고 모두 `reviewed: true`일 것을 요구한다. API를 수정하지 않기 위해 첨부 시 선택한 최종 번역을 세 필수 슬롯에 동일하게 투영한다.

예를 들어 English를 첨부하면 다음처럼 저장한다.

- `en`: English 최종 제목과 본문
- `zh`: English 최종 제목과 본문
- `vi`: English 최종 제목과 본문
- 세 항목의 `reviewed`: `true`
- 세 항목의 `sourceHash`: 최종 공지 제목과 본문 기준
- 세 항목의 `langLabel`: `English`

`lang` 값은 기존 API 계약을 위해 `en`, `zh`, `vi`를 유지한다. `langLabel`을 선택 언어명으로 통일해 별도 타입 없이 최종 첨부 언어를 복원한다.

이 투영으로 모든 수신자는 선호 언어와 무관하게 관리자가 첨부한 한 언어의 동일한 최종 공지를 받는다.

## 발송 전 검토 화면

- `D21 주요 언어 번역 미리보기`를 `최종 첨부 번역`으로 변경한다.
- 중복된 세 번역 카드를 표시하지 않는다.
- 첫 번째 호환 번역의 `langLabel`을 사용해 `최종 언어: English`처럼 표시한다.
- 최종 공지 제목과 본문 한 세트만 표시한다.
- `번역 검수`, `주요 언어 검수 완료` 문구를 제거한다.
- 기존 `승인하고 발송` 서버 액션과 API 호출은 변경하지 않는다.

## 오류 및 상태 처리

- 번역 제목이나 본문이 비어 있으면 `첨부하기`를 표시하지 않는다.
- 번역 실패 시 카드가 펼쳐진 상태를 유지하고 기존 오류 메시지를 표시한다.
- 첨부 없이 검토 진행을 누르면 `번역 후 첨부할 언어를 선택해 주세요.`를 표시한다.
- 첨부 후 왼쪽 제목이나 본문을 수정하면 `첨부한 번역 내용이 변경되었습니다. 다시 번역해 첨부해 주세요.`를 표시할 수 있는 미첨부 상태로 전환한다.
- 저장된 호환 초안은 세 번역의 제목·본문·`langLabel`이 같고 모두 검수 상태이면 첨부된 초안으로 복원한다.

## 테스트

- 한 언어만 번역한 후 `첨부하기`가 활성화되는지 확인한다.
- 첨부 시 왼쪽 제목과 본문이 선택 번역으로 교체되는지 확인한다.
- 첨부 시 세 필수 언어 슬롯이 동일한 내용, 동일한 `langLabel`, `reviewed: true`로 생성되는지 확인한다.
- 체크박스와 `검수 완료` 문구가 `/01`에서 제거됐는지 확인한다.
- 첨부하지 않은 긴급 공지가 검토 단계로 진행되지 않는지 확인한다.
- `/02`가 최종 언어와 최종 내용 한 세트만 표시하는지 확인한다.
- 기존 발송 API 호출이 변경되지 않았는지 확인한다.
- web 단위 테스트, Next.js 빌드, Docker 브라우저에서 한 언어 번역부터 검토 화면까지 확인한다.

## 기술 부채

세 필수 언어 슬롯에 같은 내용을 저장하는 것은 기존 서버 계약을 유지하기 위한 호환 계층이다. 데이터만 보면 실제로 세 언어가 번역된 것처럼 보일 수 있다. 향후 API 수정 범위가 허용되면 `attachedLang`과 단일 최종 콘텐츠 모델을 추가하고 호환 투영을 제거해야 한다.
