# 관리자 매물 상세 팝업 설계

## 목표

관리자 매물 목록에서 매물 카드를 클릭하면 동일한 관리자 워크스페이스 안에 상세 팝업을 열고, 기본 정보 수정과 매물 내리기를 안전하게 수행할 수 있게 한다.

## 범위

- 매물 카드 전체를 클릭 가능한 요소로 변경한다.
- 카드 클릭 시 접근 가능한 상세 `<dialog>`를 연다.
- 상세 조회, 기본 정보 수정, 매물 내리기 확인 상태를 같은 팝업에서 제공한다.
- 기존 `/api/trade/listings/:listingId` PATCH·DELETE 경로를 사용한다.
- 사진과 3D 도면 데이터는 수정 요청에 포함하지 않아 기존 값을 보존한다.
- API와 인프라 설정은 변경하지 않는다.

## 상세 팝업

조회 상태에서는 다음 정보를 표시한다.

- 대표 사진과 전체 사진 수
- 3D 도면 연결 여부
- 노출 상태
- 매물명
- 전체 주소
- 방 유형
- 거래 유형
- 보증금과 월세
- 매물 설명

팝업은 닫기 버튼, Esc, 바깥 영역 클릭으로 닫을 수 있다. 매물 카드는 버튼 의미와 키보드 포커스를 제공해 Enter 또는 Space로도 팝업을 열 수 있게 한다.

## 수정 흐름

`수정` 버튼을 누르면 같은 팝업이 편집 상태로 전환된다. 다음 기본 정보만 편집한다.

- 매물명
- 기본 주소
- 상세 주소
- 방 유형
- 거래 유형
- 보증금
- 월세
- 설명

저장 시 `PATCH /api/trade/listings/:listingId`로 위 필드만 전송한다. `images`와 `floorPlan` 키는 요청에 넣지 않는다. 서버의 부분 수정 계약에 따라 기존 사진과 3D 도면은 유지된다.

저장 성공 시 응답을 화면 모델로 다시 변환해 목록 카드와 팝업을 즉시 갱신하고 조회 상태로 돌아간다. 취소하면 서버 요청 없이 수정 전 값으로 돌아간다. 저장 중에는 중복 제출을 막고, 실패하면 팝업 안에 오류 문구를 표시한다.

## 매물 내리기 흐름

`매물 내리기`를 누르면 팝업 안에 별도의 확인 상태를 표시한다. 이 상태는 다음 내용을 명확히 알린다.

- 매물이 목록과 공개 노출에서 제거됨
- 작업을 되돌릴 수 없음
- 기존 문의 대화 기록은 유지됨

사용자가 최종 확인하면 `DELETE /api/trade/listings/:listingId`를 호출한다. 성공 시 팝업을 닫고 해당 카드를 목록에서 제거한다. 마지막 매물을 내리면 기존 빈 상태 화면을 표시한다. 실패하면 팝업을 유지하고 오류를 표시한다.

## 컴포넌트와 상태

서버 페이지는 현재처럼 로그인한 집주인의 매물을 조회하고 화면 모델을 만든다. 상호작용은 새 클라이언트 컴포넌트가 담당한다.

- `ManagerListingBoard`: 목록, 선택된 매물, 팝업 열림 상태, 수정 상태, 삭제 확인 상태를 관리한다.
- `ManagerListingDialog`: 상세 조회·수정·삭제 확인 UI를 렌더링한다.
- `manager-listing-model`: 상세 조회와 수정에 필요한 원본 기본 필드를 화면 모델에 포함하고 API 응답 변환을 담당한다.
- `manager-listing-api`: PATCH·DELETE 요청과 오류 메시지 정규화를 담당한다.

목록 상태는 서버에서 받은 초기값으로 시작하고, 성공한 수정·삭제 응답에 따라 클라이언트에서 갱신한다.

## 오류 처리

- 수정 실패: 입력값과 수정 상태를 유지하고 재시도 가능한 오류 표시
- 삭제 실패: 삭제 확인 상태와 팝업을 유지하고 오류 표시
- 선택된 매물이 삭제된 후: 팝업을 닫고 목록에서 제거
- API 오류 중: 관련 버튼 비활성화와 진행 문구 표시
- 사진 없음: 기존 카드와 동일한 빈 미디어 영역 표시
- 설명 없음: `등록된 설명이 없습니다.` 표시

## 접근성

- 카드 클릭 요소에 매물명을 포함한 접근 가능한 이름 제공
- `<dialog>`에 제목을 연결하는 `aria-labelledby` 제공
- 닫기·수정·취소·저장·매물 내리기·최종 확인 버튼에 명확한 이름 제공
- 네이티브 dialog의 포커스 이동과 Esc 닫기를 사용
- 파괴적 최종 확인은 일반 버튼과 시각적으로 구분하되 공유 토큰만 사용

## 테스트

구현 전에 다음 실패 테스트를 추가한다.

1. 카드 클릭과 키보드 조작으로 상세 dialog가 열린다.
2. 상세 팝업에 사진·3D·상태·주소·방 유형·가격·설명이 표시된다.
3. 수정 저장 요청은 기본 정보만 보내고 `images`·`floorPlan`을 보내지 않는다.
4. 수정 성공 시 목록 카드와 팝업 정보가 갱신된다.
5. 수정 실패 시 입력과 팝업이 유지되고 오류가 표시된다.
6. 매물 내리기는 확인 단계 전에는 DELETE하지 않는다.
7. 삭제 성공 시 팝업과 목록에서 매물이 제거되고 마지막 항목이면 빈 상태가 나타난다.
8. 삭제 실패 시 팝업이 유지되고 오류가 표시된다.
9. dialog에 닫기 버튼, Esc, 바깥 영역 닫기가 연결된다.

대상 테스트 통과 후 전체 web 테스트, web 빌드, `bash scripts/verify.sh`, Docker 브라우저 검증을 실행한다.

## 완료 기준

- 매물 카드를 클릭하면 관리자 셸 위에 상세 팝업이 열린다.
- 기본 정보를 수정하고 목록에 즉시 반영할 수 있다.
- 수정 후에도 기존 사진과 3D 도면이 보존된다.
- 확인을 거쳐 매물을 내리고 목록에서 즉시 제거할 수 있다.
- 오류와 진행 상태가 팝업 안에서 명확히 표시된다.
- 테스트와 실제 Docker 화면 검증이 통과한다.
