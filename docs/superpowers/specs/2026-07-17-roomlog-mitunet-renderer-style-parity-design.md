# RoomLog MitUNet 3D 스타일 동기화 설계

## 목표

MitUNet 편집기의 가구 배치 단계와 RoomLog 저장 후 3D 미리보기가 같은 재질과 조명으로 보이게 한다. 단, 외부 바닥은 MitUNet의 고정 `60 x 60` 크기를 그대로 사용하지 않고 도면 크기에 맞춰 동적으로 줄여서, 넓은 빈 바닥 때문에 건물이 작아 보이지 않게 한다.

이번 변경은 표시 계층만 다룬다. 저장 payload, 벽·문·창문 탐지 결과, 폴리곤 좌표, 높이, 개구부 형상과 카메라 등장 애니메이션은 변경하지 않는다.

## 선택한 방식

RoomLog의 `RoomlogThreeFloorPlanView`가 MitUNet 뷰어의 렌더링 규칙을 직접 재현한다. 저장 데이터에는 재질이나 텍스처를 추가하지 않는다.

이 방식을 선택하는 이유는 다음과 같다.

- 재질은 도면 데이터가 아니라 표시 정책이므로 저장 payload와 분리한다.
- 기존에 저장된 MitUNet 도면도 별도 마이그레이션 없이 같은 스타일로 표시할 수 있다.
- 벽 탐지와 3D 형상 생성 코드를 건드리지 않고 렌더러만 변경할 수 있다.
- 텍스처 이미지 파일을 저장하거나 전송하지 않아도 브라우저에서 같은 절차적 텍스처를 만들 수 있다.

고정 크기 바닥은 도면마다 빈 공간 비율이 달라지고, 바닥 크기를 저장 데이터에 넣는 방식은 표시 정책을 데이터에 섞으므로 사용하지 않는다.

## 렌더링 규칙

### 배경과 조명

RoomLog 3D에 MitUNet 뷰어와 같은 렌더링 설정을 적용한다.

- 옅은 푸른색 배경
- ACES Filmic 톤 매핑과 동일한 노출
- `RoomEnvironment` 기반 환경광
- 반구광, 주 방향광, 약한 주변광, 아래쪽 보조광
- 부드러운 그림자

카메라 자동 맞춤은 기존처럼 벽 경계만 사용한다. 외부 바닥 크기는 카메라 거리 계산에 포함하지 않는다.

### 벽과 창문

- 벽 측면은 무광 흰색 `MeshStandardMaterial`을 사용한다.
- 벽 상단 면은 검은색 재질을 사용한다.
- 창문은 MitUNet과 같은 불투명도, 거칠기, 투과율과 굴절률을 가진 `MeshPhysicalMaterial`을 사용한다.
- 문은 현재처럼 완전히 열린 공간으로 유지한다.
- 벽·문·창문의 폴리곤, 높이와 형상 생성 로직은 변경하지 않는다.

### 내부 우드 바닥

MitUNet의 내부 마스크 계산과 절차적 우드 패턴을 RoomLog용 순수 도우미로 옮긴다.

- 저장된 `canvasSize`와 벽·문·창문 폴리곤으로 실내 마스크를 만든다.
- 실내로 판정된 픽셀에만 따뜻한 우드 판재 패턴을 그린다.
- 판재 간 색상 변화와 이음선을 MitUNet과 같은 계산식으로 생성한다.
- 우드 바닥은 벽 아래에 위치하며 외부 콘크리트 위로 아주 조금 올려 깊이 충돌을 방지한다.

텍스처 캔버스를 만들 수 없는 예외 상황에서는 동일 계열의 단색 우드 재질로 대체해 3D 전체가 사라지지 않게 한다.

### 외부 콘크리트 바닥 크기

외부 바닥은 도면 벽 경계를 기준으로 계산한다.

```text
padding = max(planWidth, planDepth) * 0.12
groundWidth = planWidth + padding * 2
groundDepth = planDepth + padding * 2
```

즉, 도면의 긴 변 길이를 기준으로 사방에 12%만 여유를 둔다. 실측 도면과 비실측 도면 모두 현재 장면 좌표로 계산하므로 같은 시각 비율을 유지한다.

외부 바닥에는 MitUNet과 같은 회색 콘크리트 노이즈와 줄눈 패턴을 사용한다. 바닥 중심은 벽 경계 중심과 일치시킨다.

## 구성 요소

### `RoomlogThreeFloorPlanView.tsx`

- 장면 배경, 톤 매핑, 환경광과 조명을 구성한다.
- 기존 `mitunetLayout`과 벽 경계를 재사용한다.
- 외부 콘크리트 바닥과 내부 우드 바닥을 렌더링한다.
- 벽과 창문에 MitUNet 대응 재질을 연결한다.
- 기존 가구 선택·이동·완료 버튼과 카메라 제어는 그대로 유지한다.

### 새 렌더링 도우미

`room-scene` 아래에 MitUNet 표면 렌더링 전용 도우미를 둔다.

- 외부 바닥 크기 계산
- 콘크리트 `CanvasTexture` 생성
- 실내 마스크와 우드 `CanvasTexture` 생성
- 생성한 텍스처의 해제 수명 관리

도우미는 벽 폴리곤을 변경하지 않고 입력을 읽기만 한다. React 컴포넌트가 재렌더되거나 도면이 바뀌면 이전 텍스처를 `dispose()`한다.

## 데이터 흐름

1. 저장된 `MitunetFloorPlan`을 현재와 동일하게 읽는다.
2. `createMitunetSceneLayout`으로 벽·문·창문 장면 좌표를 계산한다.
3. 같은 payload의 캔버스 크기와 폴리곤으로 내부 우드 마스크를 만든다.
4. 장면 좌표의 벽 경계로 동적 콘크리트 바닥 크기를 계산한다.
5. RoomLog 렌더러가 형상과 표면 재질을 합쳐 표시한다.

저장 형식과 API 요청은 변경하지 않는다.

## 검증

### 자동 테스트

- 도면 경계가 `20 x 10`이면 외부 바닥이 `24.8 x 14.8`로 계산되는지 검증한다.
- 서로 다른 실측 비율에서도 12% 여백 규칙이 유지되는지 검증한다.
- 재질 상수, 검은 벽 상단, 물리 기반 창문과 MitUNet 대응 조명 구성이 유지되는지 회귀 테스트한다.
- 기존 문 개방, 창문 하단·상단 벽과 벽 높이 테스트를 그대로 통과시킨다.

### 화면 검증

- 같은 도면을 MitUNet 가구 배치 단계와 RoomLog 저장 후 미리보기에서 비교한다.
- 배경, 벽, 벽 상단, 창문, 우드 바닥과 콘크리트가 같은 인상인지 확인한다.
- 저장 전후 벽·문·창문 개수와 위치가 바뀌지 않는지 확인한다.
- 넓은 콘크리트 빈 공간 때문에 건물이 작아 보이지 않는지 확인한다.
- 등록 미리보기와 매물 상세 3D가 동일한 공용 렌더러를 사용하는지 확인한다.

## 비목표

- MitUNet 모델 추론, 후처리와 벽 탐지 변경
- 벽·문·창문 형상 또는 높이 변경
- 저장 payload와 데이터베이스 스키마 변경
- 가구 GLB 재질 덮어쓰기
- 가구 배치 조작 방식 변경
- 카메라 시작 위치와 등장 애니메이션 변경
