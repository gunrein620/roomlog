# 매물 등록 3D 도면 화면 전환 설계

## 목표

매물 등록의 3D 도면 흐름에서 `2D`, `3D`, `가구 배치`를 하단 토글로 자유롭게 왕복한다. 가구 배치 화면에서 2D 또는 3D로 이동할 때 현재 확정된 가구 배치를 자동 저장하고, 다시 가구 배치로 들어오면 같은 상태를 복원한다.

## 현재 문제와 원인

RoomLog 연동 흐름에서 `가구 배치`를 누르면 MitUNet 도면 뷰어에서 별도 `/floor-plan-3d/owner-furniture` 페이지로 이동한다. 이 전용 페이지에는 기존 하단 3분할 토글이 없고, `저장하고 나오기`는 매물 등록 화면으로만 이동한다. 따라서 가구 배치 이후 같은 도면의 2D 또는 3D 편집 화면으로 돌아갈 수 없다.

## 사용자 흐름

1. 도면 화면의 하단 토글은 `2D · 3D · 가구 배치` 세 항목을 유지한다.
2. 가구 배치 화면에도 동일한 형태와 위치의 하단 토글을 표시하고 `가구 배치`를 활성 상태로 표시한다.
3. 가구 배치 화면에서 `2D` 또는 `3D`를 누르면 현재 확정된 가구 배치를 요청 ID별 초안에 자동 저장한다.
4. 저장 직후 같은 요청 ID의 MitUNet 도면 편집기로 이동하고, 사용자가 선택한 2D 또는 3D 화면을 연다.
5. 이후 `가구 배치`로 다시 이동하면 저장한 가구 배치를 복원한다.
6. `저장하고 나오기`는 기존처럼 최종 도면 스냅샷을 저장하고 매물 등록으로 돌아간다.

## 상태와 전환 계약

- 전환 단위는 매물 등록이 만든 `requestId`다. 다른 매물의 도면이나 가구 초안과 섞이지 않는다.
- 가구 초안에는 도면 복귀에 필요한 MitUNet 원본 편집 상태와 확정된 가구 배열을 함께 보존한다.
- 가구를 집어 이동 중이거나 신규 가구가 아직 확정되지 않은 상태에서 전환하면 해당 미확정 조작은 저장하지 않는다. 기존 가구를 이동 중이었다면 마지막 확정 위치를 유지한다.
- 목적 화면(`2d` 또는 `3d`)은 복귀 URL에 명시하여 새로고침이나 브라우저 뒤로가기 캐시에 의존하지 않는다.
- 자동 저장 실패 시 페이지를 이동하지 않고 오류를 표시하여 작업 손실을 막는다.

## 구현 경계

- `OwnerFurnitureSimulation`이 하단 토글과 전환 목적지를 관리한다.
- 공유 `ListingTourRoom3D`는 부모가 현재 확정 가구 배열을 저장하도록 요청할 수 있는 명시적 콜백/참조 계약을 제공한다.
- RoomLog–MitUNet handoff는 요청 ID별 복귀 가능한 원본 편집 스냅샷을 저장한다.
- MitUNet 뷰어는 복귀 쿼리와 저장된 스냅샷을 읽어 도면을 복원한 뒤 요청된 2D 또는 3D 상태를 선택한다.
- 기존 일반 MitUNet 데모 흐름과 임차인용 3D 뷰어 동작은 변경하지 않는다.

## 접근성 및 화면 규칙

- 토글은 `tablist`/`tab` 의미와 `aria-selected` 활성 상태를 제공한다.
- 키보드 포커스 표시를 유지하고 각 버튼은 실제 버튼 요소를 사용한다.
- 스타일은 기존 RoomLog 토큰만 사용하고 raw hex를 추가하지 않는다.
- 하단 중앙 배치와 3분할 pill 형태는 기존 MitUNet 토글과 시각적으로 일치시킨다.

## 오류 처리

- 요청 ID 또는 초안이 없으면 기존 오류 화면과 매물 등록 복귀 링크를 유지한다.
- 자동 저장 또는 복귀 상태 직렬화가 실패하면 오류 메시지를 표시하고 현재 가구 화면에 머문다.
- 저장된 도면 복귀 스냅샷이 손상된 경우 가짜 빈 도면을 표시하지 않고 파일을 다시 선택하라는 기존 빈 상태로 폴백한다.

## 테스트

- 가구 페이지가 `2D · 3D · 가구 배치` 토글과 접근성 상태를 렌더하는 회귀 테스트를 추가한다.
- 2D/3D 전환이 현재 가구 저장 요청을 먼저 실행하고, 요청 ID와 목적 화면을 포함한 도면 URL로 이동하는지 검증한다.
- handoff가 복귀용 도면 편집 스냅샷을 요청 ID별로 보존하는지 검증한다.
- MitUNet 뷰어가 복귀 스냅샷을 복원하고 요청된 2D/3D 화면을 선택하는지 검증한다.
- 기존 웹 단위 테스트와 `bash scripts/verify.sh`를 실행한다.
