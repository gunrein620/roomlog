# 관리자 민원/하자 관리 메뉴 통합 설계

## 목표

관리자 워크스페이스의 `민원·하자` 하위 메뉴를 다음 두 항목으로 정리한다.

- `민원 대시보드`: 기존 통계형 대시보드와 주소를 그대로 유지한다.
- `민원/하자 관리`: 기존 `민원 대응`과 `하자 관리`를 하나로 합치고, 민원과 하자 티켓을 모두 표시하는 관리 테이블을 연다.

## 내비게이션

- `민원 대시보드`는 `/manager/ticket/dash/00`을 사용한다.
- `민원/하자 관리`는 `/manager/ticket/dash/00?view=management`를 사용한다.
- `view=management`는 통계 대시보드와 전체 관리 테이블을 명시적으로 구분한다.
- 기존 `type=complaint`와 `type=defect` 주소는 직접 접근 호환성을 유지하고 각각 필터된 관리 테이블을 연다.

## 화면 동작

- 파라미터가 없으면 기존 `ComplaintDashboard`를 렌더링한다.
- `view=management`이면 `ManagerDefectDashboard`를 전체 유형 필터로 렌더링한다.
- `type=complaint` 또는 `type=defect`이면 기존처럼 해당 유형으로 필터된 `ManagerDefectDashboard`를 렌더링한다.
- 통합 관리 화면 안의 기존 유형 필터는 유지하여 사용자가 전체, 민원, 하자를 다시 좁힐 수 있게 한다.

## 변경 범위

- 관리자 내비게이션 항목과 활성 상태 판정
- 민원·하자 대시보드 페이지의 화면 선택 로직
- 위 동작을 고정하는 web 단위 테스트

API, 공유 도메인 타입, 데이터 저장 방식, Docker 및 배포 설정은 변경하지 않는다.

## 검증

1. 내비게이션 계약 테스트에서 하위 메뉴가 정확히 두 개이고 통합 관리 링크가 올바른지 확인한다.
2. 페이지 선택 로직 테스트에서 기본 대시보드, 통합 관리, 기존 유형별 직접 접근을 확인한다.
3. 관련 web 테스트를 실행한다.
4. `bash scripts/verify.sh`로 기본 검증을 실행한다.
5. 검증 통과 후 기능 구현 커밋을 원격 `kms-complaint1`에 푸시한다.
