# 소통 허브 더미 대화 제거 설계

## 목표

관리인 소통 허브에서 `테스트 건물1~3`으로 표시되는 더미 대화 3건을 제거한다. 실제 대화 목록을 조회하지 못하거나 실제 대화가 없을 때 가짜 데이터를 보여주지 않고 빈 상태를 표시한다.

## 범위

- `messaging-manager-api.ts`의 `DEMO_MANAGER_THREADS`를 삭제한다.
- 더미 대화에서 파생된 `DEMO_MANAGER_THREAD_ID`를 삭제한다.
- `listManagerThreads`의 API 오류 fallback을 빈 배열로 변경한다.
- context 필터가 있는 목록 요청도 API 오류 시 빈 배열을 반환한다.
- 기존 소통 허브 빈 상태 문구인 `이 건물에는 아직 시작된 대화가 없습니다.`를 그대로 사용한다.
- 테스트 건물명, 더미 스레드 ID, 더미 대화 본문이 production API 클라이언트에 남지 않는 회귀 테스트를 추가한다.

## 제외 범위

- 공지 작성·번역·발송 결과 화면의 데모 데이터 제거
- API 또는 데이터베이스에 저장된 실제 대화 삭제
- 대화 삭제 mutation 동작 변경
- Docker, 배포 환경 변수, 네트워크 설정 변경

## 데이터 흐름

1. 소통 허브가 `listManagerThreads`로 실제 관리인 대화 목록을 요청한다.
2. 요청이 성공하면 API 응답만 화면에 표시한다.
3. 요청이 실패하면 빈 배열을 반환한다.
4. 페이지는 빈 배열을 받아 기존 빈 상태 카드를 렌더링한다.

## 오류 및 사용자 경험

- API 실패 시 가짜 대화로 정상처럼 위장하지 않는다.
- 기존 서버 로그 경고는 유지해 운영자가 조회 실패를 확인할 수 있게 한다.
- 실제 대화와 API 실패가 모두 같은 빈 상태로 보이는 현재 UI 계약은 유지한다. 별도 오류 UI 추가는 이번 범위에 포함하지 않는다.

## 테스트 및 검증

- 더미 대화 상수와 테스트 건물 문자열이 production API 클라이언트에 존재하지 않는 실패 테스트를 먼저 작성한다.
- `listManagerThreads`가 빈 fallback을 사용한다는 소스 계약을 검증한다.
- 대상 테스트와 `pnpm test:web`을 통과시킨다.
- `bash scripts/verify.sh`로 types·ui·web·api 빌드와 API 스모크를 검증한다.

## 인프라 영향

없다. web API 클라이언트와 테스트만 수정하며 인프라 파일은 변경하지 않는다.
