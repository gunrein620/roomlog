# 세입자 AI 민원 초안 피드백 개선

## 목표

세입자가 AI 비서에 민원 내용을 전송한 뒤 초안 작성 화면으로 이동하기 전까지, 시스템이 실제로 처리 중임을 일반 AI 말풍선으로 명확히 알린다. 작성 화면의 본문은 항목별로 구분해 빠르게 읽고 수정할 수 있게 한다.

## 동작

1. 세입자가 민원 내용을 전송하면 AI 대화에 다음 안내를 즉시 추가한다.
   - `말씀해 주신 내용으로 접수 초안을 작성하겠습니다. 잠시만 기다려 주세요.`
2. 별도의 인위적인 대기 시간은 추가하지 않는다. 서버가 초안을 반환하면 기존처럼 신규 요청 작성 화면을 연다.
3. 초안 전환 안내가 표시되는 동안 기존 노란 처리 문구는 숨겨 중복 피드백을 피한다.
4. 본문은 값이 있는 항목만 다음 순서로 구성한다.
   - `[문제 내용]`: AI가 생성한 요약
   - `[세부 유형]`: AI가 분류한 상세 유형 또는 요청 유형
   - `[요청 사항]`: 관리자에게 확인 및 점검을 요청하는 문장
5. AI가 파악하지 못한 발생 일시, 위치, 위험 여부 등은 임의로 만들어 넣지 않는다. 별도 폼 필드는 기존 값을 그대로 사용한다.

## 변경 범위

- `useTenantAiAssistant`: 전송 직후 일반 AI 안내 메시지 추가
- `TenantMyPage`: AI 초안을 폼 본문 형식으로 변환하는 순수 함수 적용
- 기존 정적 회귀 테스트: 안내 메시지와 구조화된 본문 연결을 검증

## 오류 처리

서버 요청이 실패하면 기존 오류 메시지를 이어서 표시한다. 작성 안내 말풍선은 사용자가 시도한 작업 기록으로 남긴다.

## 검증

- 테스트가 안내 메시지와 구조화 포맷을 요구하도록 먼저 실패하는지 확인한다.
- 최소 구현 후 대상 테스트와 web 빌드를 실행한다.
- 별도 접수 버튼이 다시 생기지 않았는지 기존 회귀 조건을 유지한다.
