# 등록자용 1인칭 가구 배치 통합 설계

## 목표

집을 등록하는 사람의 가구 배치 단계도 매물 상세의 `3D 시뮬레이션 > 가구 배치`와 같은 조작 흐름을 사용한다. 도면 추출·벽 검토·2D/3D/Floor 전체보기와 최종 등록 저장은 등록자 전용 기능으로 유지하고, 실제 가구를 선택하고 옮기는 단계만 공용 1인칭 가구 시뮬레이션으로 전환한다.

성공 기준은 다음과 같다.

- 등록자와 방문자가 동일한 단축키와 자동 표면 인식을 사용한다.
- 등록자가 만든 가구 배치가 기존 `roomlogListingFloorPlan3D:<requestId>` 스냅샷에 저장되어 매물 상세에 그대로 표시된다.
- 등록자의 도면 편집 및 저장 기능은 1인칭 모드와 충돌하지 않는다.
- 이후 가구 조작을 변경할 때 두 화면을 별도로 수정하지 않는다.

## 검토한 접근

### 1. MitUNet HTML 뷰어에 새 조작을 다시 구현

현재 `services/mitunet/viewer/index.html`의 OrbitControls 기반 배치를 WASD, 포인터 잠금, `1/2/3/E/Q/R` 방식으로 고친다. 초기 변경 범위는 작지만 매물 상세의 React/R3F 컨트롤러와 계속 두 벌로 유지된다. 자동 표면 인식, 충돌 판정, 단축키가 이후 다시 달라질 가능성이 높아 채택하지 않는다.

### 2. MitUNet 편집기 전체를 React 편집기로 교체

도면 업로드부터 추출·편집·가구 배치까지 `RoomlogFloorPlanEditor`로 즉시 통합한다. 장기적으로 단순하지만 현재 MitUNet의 도면 검토와 바닥 재질 기능까지 한 번에 이전해야 하므로 이번 요구보다 범위와 회귀 위험이 크다.

### 3. 도면 편집과 가구 시뮬레이션을 단계로 분리하고 가구 단계 공유 — 채택

MitUNet은 도면 생성과 전체보기 편집을 담당한다. 등록자가 `가구 배치`를 선택하면 현재 도면과 가구 초안을 요청 ID 기반 저장소에 넘기고, 매물 상세와 같은 React/R3F 가구 시뮬레이션 셸을 연다. `전체보기`로 돌아오면 도면 편집 화면을 복원하고, 최종 저장 시 동일한 스냅샷에 도면과 가구를 함께 기록한다.

이 방식은 기존 도면 기능을 보존하면서 사용자에게 보이는 가구 조작과 내부 상태 전이를 한 벌로 만든다.

## 사용자 흐름

1. 등록자는 기존과 같이 도면을 올리고 2D/3D/Floor 전체보기에서 벽과 바닥을 확인한다.
2. `가구 배치`를 누르면 1인칭 모드로 전환되고 포인터 잠금 안내와 초록색 중앙 조준선이 나타난다.
3. 등록자는 아래 공용 조작을 사용한다.
   - `W/A/S/D`: 이동
   - 마우스: 시야 회전
   - `2`: 포인터 잠금을 풀고 가구 카탈로그 열기 또는 닫기
   - `E`: 조준 중인 기존 가구 집기
   - `1`, `3`: 집고 있는 가구를 왼쪽·오른쪽으로 90도 회전
   - `Q`: 유효한 위치에 배치 확정
   - `R`: 기존 가구 제거, 신규 카탈로그 가구는 배치 취소
   - `Esc`: 이동 중인 기존 가구를 원위치로 되돌리거나 신규 배치 취소
4. 조준한 표면에 따라 바닥, 가구 위, 벽걸이 배치를 자동 선택한다. 유효하면 조준선이 초록색, 유효하지 않으면 빨간색으로 표시된다.
5. `전체보기`를 누르면 진행 중인 가구는 취소 또는 원복한 뒤 등록자 도면 전체보기로 돌아간다.
6. `3D 도면 저장하기`를 누르면 도면과 확정된 가구를 함께 저장하고 매물 등록 화면으로 복귀한다.

모바일 또는 coarse pointer 환경은 포인터 잠금 기반 1인칭 편집을 열지 않고 기존 전체보기와 명시적 안내를 제공한다. 이번 범위에서 모바일 전용 가구 배치 조작은 추가하지 않는다.

## 구성 요소 경계

### 공용 가구 시뮬레이션 셸

현재 `ListingTourRoom3D`에 결합된 가구 편집 상태를 재사용 가능한 컨트롤러/훅과 UI 셸로 분리한다. 공용 영역은 다음을 소유한다.

- Explore, Select, Carry 상태 전이
- 포인터 잠금과 WASD·마우스 입력
- `1/2/3/E/Q/R/Esc` 단축키 라우팅
- 중앙 레이캐스트와 자동 표면 판정
- 회전, 확정, 취소, 제거와 받침 가구 종속 이동
- 카탈로그 패널, 조준선, 유효성 피드백

`FurnitureFirstPersonControls`, `surface-placement.ts`, `RoomlogThreeFloorPlanView`의 기존 순수 계산과 렌더링 계약을 유지한다. 등록자 화면과 매물 상세는 초기 데이터와 저장 콜백만 다르게 제공한다.

### 등록자 어댑터

등록자 어댑터는 다음만 담당한다.

- MitUNet 도면과 요청 ID 기반 가구 초안 불러오기
- 1인칭 진입 전 현재 도면 상태 보존
- 확정된 `PlacedFurniture[]`를 등록 초안에 반영
- 전체보기 복귀와 최종 `3D 도면 저장하기` 처리
- 저장 오류 및 손상된 초안 복구 안내

도면 벽 수정, 바닥 재질, JSON 내려받기와 등록 화면 복귀는 공용 가구 셸로 옮기지 않는다.

### 매물 상세 어댑터

기존 매물 상세는 매물 도면과 저장된 가구 배치를 초기값으로 전달하고 브라우저별 사용자 배치를 저장한다. 등록자 전용 버튼이나 요청 ID 저장소를 알지 못한다.

## 데이터 흐름과 저장

등록 흐름의 기준 키는 기존 `roomlogListingFloorPlan3D:<requestId>`를 유지한다. 중간 전환에는 같은 요청 ID의 작업 초안을 사용하고, 최종 저장에는 다음을 포함한다.

- MitUNet 도면과 스케일 정보
- 확정된 가구 위치, 회전, 크기, 모델 정보
- 선택적 `placement` 메타데이터
  - 바닥: `{ mode: "floor" }`
  - 가구 위: `{ mode: "surface", supportFurnitureId }`
  - 벽: `{ mode: "wall", wallId }`

기존 저장값에 `placement`가 없으면 바닥 배치로 해석한다. 등록자가 가구 시뮬레이션을 열지 않고 저장해도 기존 가구와 도면은 손실되지 않는다. 다시 편집할 때는 저장된 가구를 불러와 같은 1인칭 모드에서 이어서 수정한다.

MitUNet의 `roomlog-integration.mjs`가 현재 위치·회전·크기만 다시 매핑하면서 `placement`를 버리므로, 최종 변환기는 선택적 메타데이터를 검증해 보존해야 한다. 임의의 참조 ID나 비정상 좌표는 저장 전에 거부한다.

## 모드 전환 규칙

- 1인칭 가구 모드에서는 도면 벽 편집 단축키와 OrbitControls를 비활성화한다.
- 카탈로그가 열리면 포인터 잠금을 풀고 이동 키 상태를 초기화한다.
- 카탈로그에서 가구를 고르면 Carry 상태로 바꾸고 캔버스 포커스를 복원한다.
- 전체보기로 나갈 때 Carry 상태가 신규 가구이면 폐기하고, 기존 가구이면 원래 변환으로 복구한다.
- 저장은 확정된 가구만 포함하며 pending 가구는 포함하지 않는다.
- 도면이 바뀌어 기존 벽 부착 위치가 무효가 되면 해당 벽걸이 가구를 자동 삭제하지 않고, 재진입 시 `배치 확인 필요` 상태로 표시한다.

## 오류 처리

- 포인터 잠금 실패: 클릭-드래그 시야 회전 폴백과 안내를 제공한다.
- 카탈로그 또는 GLB 로딩 실패: 기존 배치는 보존하고 해당 항목만 재시도할 수 있게 한다.
- 유효하지 않은 표면: 마지막 유효 변환을 유지하고 `Q` 저장을 막는다.
- 중간 초안 저장 실패: 전체보기로 전환하지 않고 현재 장면을 유지한다.
- 최종 저장 실패: 등록 화면으로 이동하지 않고 재시도 메시지를 표시한다.
- 손상된 가구 한 건은 격리하며, 유효한 도면과 나머지 가구는 계속 복원한다.

## 테스트와 검증

### 자동 테스트

- 공용 컨트롤러가 등록자와 매물 상세 어댑터에서 같은 상태 전이를 생성하는지 확인한다.
- 1인칭 진입·전체보기 복귀 시 pending 신규 가구 취소와 기존 가구 원복을 확인한다.
- 등록 스냅샷 왕복 후 `placement`, 위치, 회전, 크기와 모델 정보가 보존되는지 확인한다.
- MitUNet 변환기가 legacy 가구와 새 표면 메타데이터를 모두 처리하는지 확인한다.
- 저장하지 않은 pending 가구가 등록 payload에 들어가지 않는지 확인한다.
- 모바일/coarse pointer에서 1인칭 모드가 잘못 활성화되지 않는지 확인한다.

### 수동 브라우저 검증

실제 등록 흐름에서 도면을 올린 후 다음을 확인한다.

1. `가구 배치` 진입과 `전체보기` 복귀
2. `2`로 카탈로그를 열어 신규 가구 선택 후 `Q` 확정
3. `E`로 기존 가구를 집고 `1/3` 회전, `R` 제거, `Esc` 원복
4. 바닥·테이블 위·벽걸이 배치와 초록색/빨간색 피드백
5. 저장 후 등록 화면 복귀 및 미리보기 반영
6. 같은 도면 재편집 시 배치 복원
7. 등록 완료 후 매물 상세 3D 시뮬레이션에서 동일한 배치 확인

기본 검증은 `pnpm test:web`과 `bash scripts/verify.sh`로 수행하고, Docker web 이미지를 재빌드해 실제 브라우저 흐름을 확인한다.

## 범위 제외

- MitUNet 도면 추출·벽 편집·바닥 재질 기능의 React 전면 이전
- 모바일용 가상 조이스틱 기반 가구 편집
- 다단 가구 적층, 물리 엔진, 자유 높이 조절
- 서버 협업 편집이나 여러 사용자의 실시간 동기화
