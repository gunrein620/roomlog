# Poly Haven 가구 탭 설계

## 목표

기존 가구 배치 화면에 `내가구 | 등록된 가구 | 폴리` 세 소스 탭을 같은 순서와 이름으로 제공한다. Poly Haven CC0 GLB 519개는 기존 등록 가구 600개와 섞지 않고 별도 카탈로그로 지연 로딩하며, 잘리지 않는 썸네일을 보고 선택해 기존 배치 흐름으로 놓을 수 있어야 한다.

## 적용 범위

- 매물 상세의 3D 가구 배치
- 임대인 매물 등록의 3D 가구 배치
- 도면 편집기의 가구 배치
- Splat 투어 가구 편집기

모든 화면은 다음 탭을 노출한다.

1. `내가구`: 임차인 계정의 메시 변환 완료 가구
2. `등록된 가구`: 현재 운영 카탈로그 600개
3. `폴리`: S3에 보관된 Poly Haven 모델 519개

기본 선택은 기존 화면의 동작을 보존한다. 임대인 공유 3D 화면은 `내가구`, 그 외 화면은 `등록된 가구`를 기본으로 한다.

## 데이터 구조

기존 `furniture/catalog.json`은 수정하지 않는다. Poly Haven 전용 데이터는 다음 경로를 쓴다.

```text
s3://woozu-static-file/furniture/polyhaven-cc0/
  catalog.json
  source-manifest.json
  *.glb
  thumbnails/*.png
```

`catalog.json`의 각 항목은 다음 정보를 가진다.

- Poly Haven asset ID와 표시 이름
- API category와 tags
- `polyhaven-cc0/<asset-id>.glb` 상대 경로
- 실제 GLB 바운딩박스 기반 `sizeMm`
- GLB 바이트 크기
- S3 썸네일 상대 경로
- Poly Haven 원문 URL
- `CC0-1.0` 라이선스

카탈로그와 썸네일은 배포 시점의 스냅샷이다. 브라우저는 Poly Haven API를 직접 호출하지 않는다. 생성 스크립트만 고유 User-Agent로 API를 호출하고, UI에는 `Poly Haven · CC0` 출처를 표시한다.

## 로딩과 캐시

웹은 `loadPolyhavenCatalog()`를 공용 로더로 사용한다. 사용자가 처음 `폴리` 탭을 선택할 때만 `polyhaven-cc0/catalog.json`을 요청하며, 같은 페이지의 후속 호출은 모듈 단위 Promise 캐시를 공유한다.

각 화면은 소스별 배열을 분리해 유지한다. 탭 전환 시 검색어, 카테고리 필터, 페이지 제한을 해당 소스의 초기값으로 되돌린다. Poly 로딩 중에는 카드 형태의 구조적 로딩 상태를 보여주고 기존 두 탭은 계속 사용할 수 있다.

## 썸네일과 카드

- 256px 정사각형 Poly Haven 썸네일을 S3에 저장한다.
- 카드 썸네일은 정사각 프레임과 `object-fit: contain`을 사용해 모델 전체가 잘리지 않게 표시한다.
- 이미지 로딩 실패 시 중립 배경과 모델 이름 첫 글자를 표시한다.
- `loading="lazy"`와 `decoding="async"`를 유지한다.
- 50MB 이상 GLB에는 `대용량` 배지와 실제 용량을 표시한다.
- 대용량 항목도 사용자가 선택할 수 있다. 선택 후 기존 GLTF 로딩·배치 흐름을 그대로 사용한다.

## 배치 동작

Poly 항목은 기존 `FurnitureCatalogItem`으로 정규화한다. `modelUrl`, `length`, `placementCapability`, `source`를 채워 기존 `createFurnitureModel`, 표면 배치, 이동·회전·삭제·저장 로직을 재사용한다.

Poly Haven category와 tags를 룸로그 카테고리로 매핑한다. 명확하지 않은 항목은 `기타`로 두며 검색에는 asset ID, 표시 이름, category, tags를 모두 포함한다.

## 오류 처리

- Poly 카탈로그 요청 실패: Poly 탭 안에 재시도 버튼과 오류 문구를 표시한다.
- 개별 썸네일 실패: 해당 카드만 문자 폴백으로 전환한다.
- 개별 GLB 실패: 기존 3D 로더 오류 경계에서 배치 실패를 알리고 다른 항목 선택은 유지한다.
- `내가구` 또는 `등록된 가구` 로딩 성공 여부는 Poly 오류와 독립적이다.
- 카탈로그 파서는 필수 치수·경로가 잘못된 항목만 제외하고 나머지는 계속 표시한다.

## 성능

- Poly 카탈로그는 탭 선택 전 네트워크 요청을 만들지 않는다.
- 최초 30개만 렌더하고 기존 `더 보기` 흐름을 사용한다.
- GLB는 선택한 모델 하나만 로드한다.
- 50MB 이상 모델은 경고하되 사용자의 전체 519개 노출 요구에 따라 제외하지 않는다.

## 검증

- 카탈로그 변환: 경로, 치수, 썸네일, 용량, 카테고리 매핑 단위 테스트
- 지연 로딩: Poly 탭 선택 전 미호출, 선택 후 1회 호출, 재시도 테스트
- UI 계약: 네 화면 모두 세 탭의 순서·이름·count·Poly 패널을 노출
- 배치: Poly 항목이 기존 배치 draft와 저장 payload에 model URL과 치수를 보존
- 썸네일: `object-fit: contain`, 문자 폴백, 대용량 배지 확인
- 운영 데이터: S3 `catalog.json` 519개, 썸네일 519개, GLB 519개 및 총 바이트 검증
- 회귀: `pnpm test:web`, 관련 집중 테스트, `bash scripts/verify.sh`

## 완료 조건

- 모든 가구 배치 화면에 `내가구 | 등록된 가구 | 폴리` 탭이 표시된다.
- Poly 탭에서 519개를 검색·필터·선택·배치할 수 있다.
- 썸네일이 잘리지 않고 실패 시 폴백이 보인다.
- 기존 등록 가구 600개 카탈로그가 유지된다.
- 변경은 `main`에 커밋하되 원격으로 푸시하지 않는다.
