# 크레딧 업체 지급 입출금 원장 연동 설계

## 목표

Gara 업체 지급으로 크레딧이 실제 차감되면 청구·수납의 입출금 내역에 같은 금액의 출금 행을 표시한다.

## 설계

`CreditLedgerEntry`를 지급 사실의 단일 원본으로 유지한다. `GaraVendorPayoutRequest`와 연결된 음수 크레딧 원장만 읽기 모델로 변환하여 기존 입출금 목록에 병합한다. 별도 `Cost` 레코드를 만들지 않으므로 재시도·새로고침에서 비용이 중복 생성되지 않는다.

출금 행에는 원장 ID, 원장 생성 시각, 절댓값 금액, 업체명, `업체 크레딧 지급` 항목명 및 `크레딧 업체 지급` 출처를 담는다. 크레딧 충전, 시작 잔액, 정정 원장은 이 화면의 업체 지급 출금에서 제외한다.

## 검증 기준

`CREDIT_DEBITED` Gara 지급과 연결된 음수 원장이 있을 때 `/manager/bills/deposits`의 `ledgerRows`에 출금 행 하나가 추가된다. 동일 원장은 한 번만 나오며, 화면의 출금 합계와 상세에 반영된다.
