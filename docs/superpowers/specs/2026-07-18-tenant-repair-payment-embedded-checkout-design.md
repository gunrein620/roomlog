# 세입자탭 수리비 결제 체크아웃 이관 설계

## 목표

세입자 직접 부담 수리비 결제를 별도 화면으로 떠나는 링크가 아니라 세입자탭 하자 상세 시트 안에서 시작한다. 기존 `/tenant/repair-payment/:paymentRequestId` 라우트와 Toss 성공·실패 콜백은 호환을 위해 유지한다.

## 화면과 상태

- `TenantVendorWorkflowPanel`의 `결제하기` 버튼은 같은 상세 시트 안에서 `TenantRepairPaymentCheckout`을 연다.
- 임베드 상태에서는 체크아웃 헤더의 뒤로가기가 링크가 아니라 닫기 버튼으로 동작한다.
- 체크아웃을 닫으면 업체 워크플로를 다시 조회해 직접결제 요청·완료 상태를 즉시 반영한다.
- 독립 라우트로 열린 체크아웃은 기존 `/living?complaintId=...` 링크 동작을 유지한다.

## Toss 복귀

- 임베드 체크아웃은 현재 `/living` 경로와 `complaintId`를 주문 `returnPath`에 저장한다.
- 성공·실패 콜백의 복귀 경로 정규화는 정확한 `/living` 경로만 허용하고 외부 URL·유사 경로는 계속 거부한다.
- `/living?complaintId=...`로 돌아오면 기존 `TenantMyPage` 로직이 해당 하자 상세 시트를 다시 연다.

## 검증

- 복귀 경로 allowlist가 `/living`의 쿼리·해시를 보존하고 외부/유사 경로를 거부한다.
- 체크아웃 컴포넌트가 임베드 닫기 콜백과 `/living` returnPath를 사용한다.
- 업체 워크플로 패널이 링크 대신 임베드 체크아웃을 열고 닫을 때 데이터를 재조회한다.
- 기존 독립 체크아웃 라우트와 콜백은 그대로 빌드된다.
