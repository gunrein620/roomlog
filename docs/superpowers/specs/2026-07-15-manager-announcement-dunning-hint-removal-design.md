# 관리자 공지 작성 연체 안내 문구 제거 설계

## 목표

관리자 `공지 작성` 화면의 대상 선택 안내에서 다음 문구를 제거한다.

> 미납 세대 옵션은 없습니다. 연체·독촉은 별도 채널에서 처리합니다.

## 변경 범위

- `AnnouncementComposer`의 대상 선택 안내에서 위 문장과 해당 줄바꿈만 삭제한다.
- `공지 대상을 선택하세요.` 문구는 유지한다.

## 유지 범위

- 공지 대상 `전체`, `건물`, `호실` 선택 기능
- 선택된 건물 및 호실 계산 로직
- 공지 초안 저장과 발송 검토 흐름
- 서버의 공지 채널 연체·독촉 문구 차단 정책
- 청구 채널의 연체·독촉 처리 흐름

## 테스트

- `AnnouncementComposer` 소스에서 제거 대상 문구가 사라졌는지 회귀 테스트로 확인한다.
- `공지 대상을 선택하세요.`와 `전체`, `건물`, `호실` 대상 선택 옵션이 유지되는지 확인한다.
- web 관련 회귀 테스트와 프로덕션 빌드를 통과시킨다.

## 제외 범위

- 미납 세대 공지 옵션 추가
- 연체·독촉 발송 정책 변경
- 공지 또는 청구 API 변경
- 대상 선택 UI 재설계
