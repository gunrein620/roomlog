# 관리자 매물 관리 목록 화면 설계

## 목표

관리자 워크스페이스의 `매물 관리` 메뉴를 눌렀을 때 매물 등록 화면으로 바로 이동하지 않고, 동일한 관리자 UI 안에서 현재 로그인한 집주인이 등록한 매물 목록을 확인할 수 있게 한다.

## 범위

- 관리자 전용 매물 목록 경로를 `/manager/listing`으로 추가한다.
- 사이드바의 `매물 관리` 링크를 `/manager/listing`으로 변경한다.
- 관리자 워크스페이스 밖으로 이동한다는 외부 링크 표시를 제거한다.
- 목록 화면 상단에 `새 매물 등록` 버튼을 제공하고 기존 `/sell` 등록 화면으로 연결한다.
- 현재 로그인한 집주인이 소유한 매물만 목록에 표시한다.
- 인프라, 배포 설정, 매물 등록 폼, API 계약은 변경하지 않는다.

## 화면 구성

목록 화면은 기존 `ManagerAppShell`을 사용한다. 상단에는 `매물 관리` 제목, 간단한 설명, `새 매물 등록` 버튼을 배치한다.

매물 목록의 각 항목에는 다음 정보를 표시한다.

- 대표 사진
- 매물명
- 주소
- 거래 유형과 금액
- 노출 상태
- 등록 사진 수
- 3D 도면 연결 여부

등록된 매물이 없으면 빈 상태 안내와 `새 매물 등록` 버튼을 함께 표시한다. API 요청에 실패하면 데모 데이터를 만들지 않고 목록을 불러오지 못했다는 상태를 표시한다.

## 데이터 흐름

1. 서버 컴포넌트에서 `requireUser("LANDLORD")`로 관리자 접근 권한과 로그인 사용자를 확인한다.
2. 기존 `serverFetch("/trade/listings")`를 사용해 매물 목록을 조회한다.
3. 응답 중 `ownerId`가 로그인 사용자의 `userId`와 같은 매물만 남긴다.
4. 화면 표시용 모델로 변환해 목록 컴포넌트에 전달한다.

새 API나 데모 폴백은 추가하지 않는다. 현재 API가 전체 매물을 반환하므로 화면에서는 기존 통합 대시보드와 동일한 소유자 필터를 적용한다.

## 라우팅과 내비게이션

- `매물 관리`: `/manager/listing`
- `새 매물 등록`: `/sell`
- `/manager/listing`에서는 사이드바의 `매물 관리`가 활성 상태여야 한다.
- `매물 관리` 항목은 더 이상 외부 링크로 표시하지 않는다.

## 오류 및 빈 상태

- 조회 성공, 매물 있음: 본인 소유 매물 목록을 표시한다.
- 조회 성공, 매물 없음: 등록된 매물이 없다는 안내와 등록 버튼을 표시한다.
- 조회 실패: 실패 안내를 표시하고 등록 버튼은 유지한다.
- 다른 집주인의 매물은 응답에 포함되어도 화면에 표시하지 않는다.

## 테스트

구현 전에 다음 동작을 검증하는 실패 테스트를 추가한다.

1. 사이드바 `매물 관리`가 `/manager/listing`으로 연결되고 외부 링크가 아니다.
2. `/manager/listing`이 `매물 관리` 내비게이션 항목을 활성화한다.
3. 표시 모델이 로그인한 집주인의 매물만 남기고 가격·사진·3D 상태를 변환한다.
4. 목록 화면에 제목, 등록 버튼, 목록 상태, 빈 상태 및 오류 상태가 존재한다.

대상 테스트 통과 후 전체 web 테스트와 기본 검증 범위에서 회귀 여부를 확인한다.

## 완료 기준

- `매물 관리` 클릭 시 관리자 셸이 유지된다.
- 등록한 매물 목록을 같은 UI에서 볼 수 있다.
- `새 매물 등록` 버튼으로 기존 등록 화면에 진입할 수 있다.
- 본인 소유가 아닌 매물은 표시되지 않는다.
- 기능 테스트와 web 회귀 테스트가 통과한다.
