# 관리자 공지 한국어 단독 발송 설계

## 목표

관리자가 `/manager/messaging/01`에서 번역 언어를 선택하거나 번역문을 첨부하지 않아도 입력한 한국어 제목과 본문을 그대로 검토하고 발송할 수 있게 한다. 긴급·생활·행사 공지 모두 같은 방식을 지원한다.

## 현재 문제

- 긴급 공지에서 `검토하고 발송으로`를 누르면 화면이 첨부 번역을 강제한다.
- 화면 검증을 통과하더라도 API는 English, 中文, Tiếng Việt 번역 세 개와 검수 완료 상태를 요구한다.
- 따라서 화면 오류만 제거하면 검토 또는 실제 발송 단계에서 다시 실패한다.

## 사용자 흐름

### 발송 언어 선택

- 공지 내용 영역에 `한국어 원문으로 발송`과 `번역본으로 발송` 중 하나를 선택할 수 있는 발송 언어 옵션을 제공한다.
- 새 공지는 `한국어 원문으로 발송`을 기본값으로 시작한다.
- 한국어 발송을 선택하면 번역 카드 사용 여부와 관계없이 현재 한국어 제목과 본문을 최종 콘텐츠로 사용한다.
- 번역본 발송을 선택하면 기존처럼 한 언어를 번역하고 `첨부하기`로 최종 콘텐츠를 정해야 한다.

### 검토 진입

- 한국어 발송에서는 제목, 본문, 발송 대상만 검증한다.
- 번역본 발송에서는 기존 `번역 후 첨부할 언어를 선택해 주세요.` 검증을 유지한다.
- 검토 화면은 최종 발송 언어를 `한국어` 또는 첨부한 번역 언어로 명확히 표시한다.

### 초안 저장과 재진입

- 한국어 단독 초안은 번역 배열을 비운 상태로 저장한다.
- 저장된 초안을 다시 열었을 때 번역이 없으면 한국어 발송 모드로 복원한다.
- 기존 단일 번역 호환 초안은 첨부 번역 발송 모드로 복원한다.

## 서버 규칙

- 긴급 공지라도 번역이 하나도 없는 경우 한국어 단독 발송으로 인정한다.
- 번역 데이터가 존재하는 긴급 공지는 기존 규칙을 유지한다.
  - English, 中文, Tiếng Việt 슬롯이 각각 정확히 하나 존재해야 한다.
  - 제목과 본문이 비어 있지 않아야 한다.
  - 현재 원문 해시와 일치해야 한다.
  - 검수 완료 상태여야 한다.
- 이 구분으로 한국어 단독 발송을 허용하면서 불완전한 번역본이 발송되는 것은 계속 차단한다.

## 데이터 및 타입 경계

- 공유 타입과 DB 스키마에 새 필드를 추가하지 않는다.
- `translations.length === 0`을 한국어 단독 발송으로 해석한다.
- 기존 단일 번역 첨부는 세 언어 슬롯에 같은 최종 내용을 투영하는 호환 형식을 유지한다.
- 한국어 단독 발송은 가짜 번역 슬롯을 만들지 않는다.

## 오류 및 상태 처리

- 한국어 발송 모드에서는 번역 미선택 오류를 표시하지 않는다.
- 번역본 발송 모드에서 첨부 번역이 없으면 기존 안내 문구를 표시한다.
- 한국어 제목 또는 본문이 비어 있으면 기존 필수 입력 오류를 표시한다.
- 발송 대상이 없으면 기존 대상 선택 오류를 표시한다.
- 번역본 발송 모드에서 원문이나 첨부 내용을 변경해 첨부 상태가 깨지면 재첨부를 요구한다.

## 테스트

### Web

- 새 공지의 기본 발송 모드가 한국어인지 확인한다.
- 긴급 공지가 번역 없이 검토 단계로 진행되는지 확인한다.
- 한국어 모드 저장 payload의 번역 배열이 비어 있는지 확인한다.
- 번역본 모드에서는 첨부 언어 선택을 계속 요구하는지 확인한다.
- 저장된 한국어 초안과 기존 첨부 번역 초안이 올바른 모드로 복원되는지 확인한다.
- 검토 화면에 최종 발송 언어가 표시되는지 확인한다.

### API

- 번역이 없는 긴급 공지가 승인·발송되는지 확인한다.
- 번역 배열이 일부만 있거나 미검수인 긴급 공지는 계속 거절되는지 확인한다.
- 생활·행사 공지의 기존 발송 동작이 유지되는지 확인한다.
- 한국어 단독 긴급 공지도 확인 게이트와 읽음/확인 분리가 유지되는지 확인한다.

### 통합 검증

- `pnpm test:web`
- `pnpm test:api`
- `bash scripts/verify.sh`
- Docker 이미지를 다시 빌드하고 `/manager/messaging/01`에서 한국어 긴급 공지 작성부터 검토·발송까지 확인한다.

- 공지 대상 건물·호실 선택 규칙
- 긴급 공지 확인 게이트
- 임차인 공지 카드의 읽음/확인 의미
- Docker, 배포, 데이터베이스 등 인프라 구성
