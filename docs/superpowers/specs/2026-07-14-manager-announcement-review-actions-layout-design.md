# 관리자 공지 검토 액션 배치 변경 설계

## 목표

관리자 공지 발송 전 검토 화면에서 보조 설명 영역을 제거하고, 검토 및 발송 액션을 최종 발송 정보와 가까운 위치에 배치한다.

## 화면 변경

- 우측 사이드 영역의 `폰 read-only 미리보기`, `문구 톤 체크`, `확인 게이트`를 제거한다.
- `체크 완료`와 발송 상태 버튼을 한 액션 그룹으로 묶는다.
- 긴급 공지에서는 액션 그룹을 `최종 발송 언어` 카드의 우측 하단에 배치한다.
- 일반 공지는 `최종 발송 언어` 카드가 없으므로 액션 그룹을 수신자 명단 카드 아래 우측에 배치한다.
- 수신자가 없으면 기존처럼 발송 버튼을 회색 `수신자 없음` 상태로 비활성화한다.
- 수신자가 있으면 기존처럼 `승인하고 발송`, 처리 중에는 `발송 중...`을 표시한다.

## 레이아웃

- 기존 본문과 우측 사이드바의 2열 구조를 제거하고 단일 열로 전환한다.
- 액션 그룹은 우측 정렬하며, 넓은 화면에서는 버튼 두 개를 가로로 배치한다.
- 좁은 화면에서는 버튼이 사용 가능한 너비에 맞춰 줄바꿈될 수 있도록 한다.
- 색상, 간격, 테두리는 기존 UI 컴포넌트와 CSS 토큰만 사용한다.

## 동작 유지

- 공지 초안 조회, 수신자 산정, 번역 표시, 발송 서버 액션은 변경하지 않는다.
- `체크 완료`는 기존과 동일한 정적 버튼으로 유지한다.
- 발송 버튼의 활성화 조건과 오류 메시지 처리는 변경하지 않는다.

## 검증

- 소스 회귀 테스트로 제거 대상 문구 세 개가 검토 화면에서 사라졌는지 확인한다.
- 긴급 공지의 최종 발송 언어 카드 안에 액션 그룹이 포함되는지 확인한다.
- 일반 공지에도 액션 그룹이 렌더링되는 구조인지 확인한다.
- 수신자 없음 상태의 비활성 버튼 문구 및 스타일 회귀 테스트를 유지한다.
- 웹 단위 테스트, 웹 프로덕션 빌드, Docker 재빌드 후 실제 화면을 확인한다.
