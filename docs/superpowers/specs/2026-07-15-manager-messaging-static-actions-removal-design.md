# 관리자 메시징 미작동 보조 액션 제거 설계

## 목표

관리자 메시징 상세 화면에서 실제 동작이 없는 오른쪽 보조 액션과 AI 답장 초안 영역을 제거해, 사용자가 실행 가능한 기능으로 오인하지 않도록 한다.

## 제거 범위

- `사진 요청` 버튼
- `설명 요청` 버튼
- `AI 답장 초안` 카드 전체
  - 안내 문구
  - `초안 적용` 버튼
- `음성 받아쓰기 → 텍스트 확인` 버튼
- 위 요소를 담던 오른쪽 보조 `aside`

## 유지 범위

- 메시지 상세 제목과 삭제 기능
- 대화 맥락 카드
- 메시지 타임라인
- 답장 입력과 `답장 보내기`
- 새 메시지 자동 갱신과 관리인 읽음 처리

## 레이아웃

오른쪽 340px 보조 열을 남기지 않는다. 상세 본문의 2열 grid를 제거하고 기존 주 콘텐츠 열을 전체 가용 너비로 확장한다. 별도 CSS 숨김 처리는 사용하지 않고 미작동 JSX를 삭제한다.

## 테스트

- 상세 페이지 소스에서 제거 대상 문구와 `StaticButton` 사용이 모두 사라졌는지 확인한다.
- 메시지 타임라인, 답장 폼, 읽음 처리 컴포넌트가 유지되는지 확인한다.
- 상세 본문에 `340px` 보조 열과 보조 `aside`가 남지 않았는지 확인한다.
- 기존 관리자 메시징 상세 회귀 테스트와 web 빌드를 통과시킨다.

## 제외 범위

- 사진·설명 요청 기능의 신규 구현
- AI 답장 생성 기능의 신규 구현
- 음성 받아쓰기 기능의 신규 구현
- 메시지 API와 읽음 배지 동작 변경
