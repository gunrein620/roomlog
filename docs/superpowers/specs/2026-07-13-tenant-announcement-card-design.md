# 세입자 마이페이지 집주인 공지 연동 설계

## 목표

관리자가 공지를 승인·발송하면 대상 세입자의 통합 마이페이지 `집주인 공지사항` 카드에 최신 공지가 표시되게 한다. 초안 저장만 한 공지는 노출하지 않으며, 대상이 아닌 세입자에게도 노출하지 않는다.

## 범위

- 기존 `GET /tenant/messaging/announcements` 응답을 세입자 통합 마이페이지에서 조회한다.
- 발송된 공지 중 `sentAt`이 가장 최신인 한 건을 카드에 표시한다.
- 공지가 없을 때, 불러오는 중일 때, 조회에 실패했을 때를 서로 다른 상태로 표시한다.
- 공지 발송 후 현재 화면을 열어둔 세입자에게도 갱신되도록 기존 `roomlog:activity` 이벤트를 이용한다.
- 공지 목록·상세 화면과 관리자 작성·발송 API의 계약은 변경하지 않는다.
- Docker, 배포, 환경변수 등 인프라 파일은 변경하지 않는다.

## 접근 방식 비교

### 채택: 마이페이지에서 기존 BFF 직접 조회

`TenantMyPage`가 브라우저 전용 BFF인 `/api/tenant/messaging/announcements`를 `cache: "no-store"`로 조회한다. 기존 httpOnly 쿠키 인증 프록시와 백엔드 수신자 필터를 그대로 사용하므로 새 API나 권한 로직이 필요 없다.

### 미채택: 서버 컴포넌트 공지 화면 재사용

기존 `/tenant/messaging/00`은 서버 컴포넌트이며 `TenantMyPage`는 대형 클라이언트 컴포넌트다. 화면 전체를 재사용하면 컴포넌트 경계와 라우팅 책임이 섞인다.

### 미채택: 홈 전용 공지 API 추가

최신 한 건만 반환하는 별도 API는 응답 크기를 줄일 수 있지만, 현재 데이터 규모에서 이점이 작고 공지 권한·표시 로직을 중복시킨다.

## 컴포넌트와 데이터 흐름

1. 관리자가 `승인하고 발송`을 누른다.
2. 기존 백엔드가 `MessagingAnnouncement`와 대상 세입자별 `MessagingAnnouncementDelivery`를 저장하고 `roomlog:activity`를 broadcast한다.
3. `TenantMyPage`는 `/api/tenant/messaging/announcements`를 조회한다.
4. 백엔드는 로그인한 세입자의 delivery가 있는 공지만 반환한다.
5. 웹의 순수 선택 함수가 `sentAt` 내림차순으로 최신 한 건을 선택한다.
6. 카드는 최신 공지의 제목, 본문 요약, 발신자, 발송 시각을 표시한다.
7. 카드를 누르면 기존 공지 상세 `/tenant/messaging/02?id=<announcementId>`로 이동한다.

## UI 상태

- `loading`: `공지사항을 확인하고 있습니다.`
- `ready`: 최신 공지 제목과 본문을 표시하고 상세 링크를 제공한다.
- `empty`: `임대인으로부터 전달된 새로운 소식이 없습니다.`
- `error`: `공지사항을 불러오지 못했습니다. 잠시 후 다시 확인해 주세요.`

API 실패를 빈 목록으로 간주하거나 데모 공지로 대체하지 않는다. 실제 공지가 없다는 의미와 시스템 오류를 구분하기 위해서다.

## 갱신 정책

- 마이페이지 마운트 시 1회 조회한다.
- `roomlog:activity` 이벤트가 오면 다시 조회한다.
- 브라우저가 다시 포커스되거나 숨김 상태에서 visible로 돌아오면 다시 조회한다.
- 컴포넌트가 unmount되면 소켓·브라우저 이벤트 리스너를 제거한다.
- 동시에 여러 조회가 완료돼도 unmount 이후 상태를 갱신하지 않는다.

## 테스트

- 순수 선택 함수: 비어 있는 목록, 최신 `sentAt` 선택, 입력 배열 비변경을 검증한다.
- 세입자 카드 계약 테스트: 실제 BFF 경로, 네 가지 상태 문구, 상세 링크 구성을 검증한다.
- 백엔드 서비스 테스트: 초안만 저장했을 때는 세입자 목록에 없고, 승인·발송 후 대상 세입자 목록에 나타나는지 검증한다.
- 회귀 테스트의 오래된 데모 수리 문구 기대값은 현재 실데이터 기반 화면에 맞게 제거한다.
- Docker가 가능한 환경에서는 관리자 발송 후 대상 세입자 `/living` 카드 갱신과 비대상 세입자 미노출을 확인한다.

## 제외 범위

- 여러 공지를 카드 안에서 슬라이드하거나 목록으로 보여주는 기능
- 읽음·확인 상태를 카드 노출만으로 변경하는 기능
- 푸시 알림과 서비스워커
- 공지 작성·번역·수신자 선정 정책 변경
