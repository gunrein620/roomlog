# 청구·수납 수금 분석·연체 목록·상호작용 설계

## 목표

청구·수납의 네 화면이 서로 다른 질문에 답하도록 역할을 분리한다.

| 화면 | 답해야 하는 질문 |
| --- | --- |
| 대시보드 | 이번 달 청구·수납·입금·연체의 전체 상황은 어떤가? |
| 수금현황 | 선택 범위의 수금 성과가 시간에 따라 어떻게 변했고 언제 수납되는가? |
| 입출금 내역 | 실제 입금과 납부 신고가 어떤 청구에 연결됐는가? |
| 연체관리 | 어떤 임차인이 얼마나 연체됐고 다음 처리 대상은 무엇인가? |

## 1. 수금현황

### 범위 선택

- 상단의 건물 선택과 청구 월을 유일한 분석 범위 선택 수단으로 사용한다.
- `전체 건물`이면 보유 건물 전체를 하나의 범위로 집계한다.
- 특정 건물이 선택되면 해당 건물만 같은 분석 구조로 보여준다.
- 본문에 건물을 다시 선택하거나 드릴다운하는 목록을 두지 않는다.
- 선택 건물은 다른 건물과 비교하지 않고 자기 과거 실적과 비교한다.

### 분석 구성

#### 수금 성과 진단

- 이번 달 수금률
- 최근 3개월 및 6개월 평균 수금률
- 각 평균 대비 이번 달 증감
- 청구액과 확정 수납액
- 완납 세대 비율
- 부분 수납 세대 비율

#### 수납 시점 분석

- 납부기한 전 수납 비율
- 월초·중순·말 확정 수납액
- 평균 수납 완료일
- 이번 달 월중 누적 수납 곡선
- 지난달 월중 누적 수납 곡선

#### 12개월 월별 성과표

- 청구월
- 청구액
- 확정 수납액
- 수금률
- 완납 세대 수
- 부분 수납 세대 수
- 전월 대비 수금률 변화

### 데이터 원칙

- 확정 수납만 수납 실적으로 집계한다.
- 확인 중 입금과 미연결 입금은 확정 수납에서 제외한다.
- 수납 시점은 기존 결제·입금 확정 기록의 날짜를 집계한다.
- 필요한 API 변경은 기존 기록을 집계해 응답 필드를 추가하는 수준으로 제한한다.
- DB 스키마와 마이그레이션은 변경하지 않는다.
- 월세와 관리비를 성과 비교 대상으로 사용하지 않는다.
- 임차인별 연체일·독촉 대상·장기 연체 분류는 수금현황에 표시하지 않는다.

## 2. 연체관리

### 목록 역할

- `독촉 대상`에는 독촉 가드를 통과해 AI 채팅에서 처리할 수 있는 청구만 표시한다.
- 납부 신고 또는 미연결 입금이 있는 청구는 `입금 확인 대기`로 분리한다.
- 각 행에는 건물·호실·임차인, 미수금, 연체일, 연체 단계를 표시한다.

### 연체 단계

- 1~7일: `초기 연체`
- 8~30일: `연체`
- 31일 이상: `장기 연체`

### 필터와 정렬

- 전체, 초기 연체, 연체, 장기 연체 필터를 제공한다.
- 건물·호실·임차인 검색을 유지한다.
- 기본 정렬은 연체일 높은 순으로 한다.
- 미수금 많은 순 정렬을 추가한다.

### 긴 목록 처리

- 데스크톱에서 왼쪽 목록은 약 6개 행이 보이는 최대 높이를 가진다.
- `독촉 대상 / 입금 확인 대기` 전환 영역은 목록 상단에 고정한다.
- 결과 행 영역만 세로로 스크롤한다.
- 목록이 최대 높이보다 짧으면 스크롤바를 표시하지 않는다.
- 오른쪽 상세는 왼쪽 목록 스크롤의 영향을 받지 않는다.
- 오른쪽 상세가 자체 높이를 넘을 때만 별도로 스크롤한다.

### 상세와 처리

- 청구월, 납부기한, 총 청구액, 확정 수납액, 미수금, 연체일, 연체 단계, 입금 확인 상태를 표시한다.
- 독촉 대상은 `AI 채팅에서 처리`로 연결한다.
- 메시지는 관리자가 확인하기 전 자동 발송하지 않는다.
- 입금 확인 대기는 입출금 내역으로 연결한다.

## 3. 청구·수납 전용 상호작용 상태

### 적용 범위

- `/manager/billing` 아래의 대시보드, 수금현황, 입출금 내역, 연체관리, 청구서 생성에만 적용한다.
- 버튼, 필터, 검색, 정렬, 선택 입력, 일반 입력, 목록 선택, 상세 펼침, 텍스트 링크를 포함한다.

### 동작

- 버튼은 hover에서 배경·테두리·글자색을 자연스럽게 강조한다.
- 입력과 선택 요소는 hover에서 테두리를 강조하고 focus에서 더 강한 포커스 링을 표시한다.
- 선택 가능한 목록 행은 hover에서 옅은 배경을 표시한다.
- 상세 펼침과 텍스트 링크는 글자색과 아이콘을 함께 강조한다.
- 위치 이동, 확대 등 레이아웃을 흔드는 효과는 사용하지 않는다.
- 비활성 요소에는 hover 효과를 적용하지 않는다.
- 클릭할 수 없는 지표와 카드는 hover 상태를 만들지 않는다.
- 키보드 사용을 위해 `:focus-visible` 상태를 hover와 구분해 제공한다.
- 모든 스타일 값은 기존 디자인 토큰을 사용한다.

## 제외 범위

- 왼쪽 사이드바와 공통 관리자 셸
- 공통 상단 네비게이션의 구조 또는 스타일 변경
- 청구·수납 외 다른 관리자 화면
- 공통 UI 컴포넌트의 전역 hover 변경
- 입출금 내역의 기능 및 정보 구조 재설계
- 월세와 관리비 비교
- 자동 독촉 발송
- DB 스키마와 마이그레이션

## 완료 조건

- 상단 건물 선택만으로 전체 또는 특정 건물 분석 범위가 바뀐다.
- 수금현황 본문에 중복 건물 선택 목록이 없다.
- 수금현황에 임차인별 연체 처리 정보가 섞이지 않는다.
- 연체관리에서 연체 단계와 연체일을 임차인별로 확인할 수 있다.
- 연체 대상이 많아져도 페이지 전체가 목록 길이만큼 늘어나지 않는다.
- 청구·수납의 실제 상호작용 요소만 일관된 hover와 focus 피드백을 제공한다.
- 관련 단위 테스트, web 빌드, Docker 재빌드가 통과한다.
