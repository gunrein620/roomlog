# MitUNet 가구 배치 단계 설계

## 목표

`/floor-plan-3d/mitunet`에서 도면 인식과 3D 구조 확인을 마친 뒤 같은 Three.js 장면에서 별도의 가구 배치 단계로 전환한다. 가구 배치 단계에서는 원본 도면 이미지를 숨기고, 실내에는 우드 바닥재를, 실외에는 도로 느낌의 바닥을 표시한다. 벽의 측면은 흰색, 상단 면은 검은색으로 표현하며 1,680개의 GLB 가구를 검색·배치·이동·회전·삭제할 수 있게 한다.

## 현재 구조와 제약

- `viewer/index.html`이 2D 검토 화면과 Three.js 3D 장면을 함께 관리한다.
- `buildInputImagePlane()`이 원본 도면을 수평 Plane으로 만들어 벽 아래에 표시한다.
- 벽, 문, 창문은 `loadPlan()`에서 동일한 좌표 변환과 카메라 프레이밍을 사용한다.
- 벽 상승 애니메이션은 `animations`와 `replayRiseAnimations()`가 담당한다.
- 가구 manifest와 GLB는 같은 출처의 `/floor-plan-3d/furniture-assets/*`에서 제공된다.
- manifest는 1,680개 항목과 각 모델의 `category`, `relativePath`, `sizeMm`을 제공한다.
- 기존 벽·문·창문 추출, 형상 생성, 카메라 전환, 상승 애니메이션 동작은 변경하지 않는다.

## 검토한 접근

### 1. 동일 장면에서 단계 전환 — 채택

현재 `scene`, `planGroup`, 카메라와 벽 Mesh를 그대로 유지하고 가구 배치용 레이어와 UI만 추가한다. 구조 확인에서 가구 배치로 전환할 때 원본 도면 Plane의 `visible`만 끄고 바닥재와 가구 레이어를 켠다.

장점은 벽 위치가 다시 계산되지 않고 기존 카메라와 애니메이션을 보존한다는 점이다. 사용자가 구조 확인과 가구 배치를 오가도 배치 상태가 유지된다.

### 2. 가구 배치 전용 Three.js 장면 재생성 — 제외

구조 확인 장면과 가구 장면을 분리할 수 있지만 좌표 변환, 벽 재생성, 카메라 동기화를 중복 구현해야 한다. 기존 벽과 새 바닥의 위치가 어긋날 위험이 크다.

### 3. 기존 RoomLog 가구 편집기로 이동 — 제외

기존 React 편집기를 재사용할 수 있지만 사용자가 요구한 “렌더링 후 같은 페이지의 다음 단계”가 되지 않는다. 현재 요청에서는 MitUNet 뷰어 안에 가구 배치 흐름을 둔다.

## 사용자 흐름

1. 사용자가 도면을 업로드하고 2D 인식 결과를 검토한다.
2. `Show 3D`를 눌러 기존 방식으로 구조를 생성하고 상승 애니메이션을 확인한다.
3. 3D 구성이 완료된 경우에만 `다음: 가구 배치` 버튼을 활성화한다.
4. 버튼을 누르면 현재 카메라와 벽은 유지하고 원본 도면 Plane을 숨긴다.
5. 가구 배치용 실내 바닥, 실외 도로 바닥, 카탈로그 패널을 표시한다.
6. 카탈로그에서 가구를 고르면 반투명 미리보기가 실내 바닥의 포인터 위치를 따라간다.
7. 실내의 유효한 위치를 클릭하면 가구가 배치된다. 실외에서는 미리보기를 붉게 표시하고 배치를 막는다.
8. 배치된 가구를 선택하면 이동, 90도 회전, 삭제할 수 있다.
9. `구조 확인`을 누르면 가구와 마감재를 숨기고 기존 3D 구조 화면으로 돌아간다. 다시 가구 배치로 전환하면 이전 배치 상태를 복원한다.

## 장면 계층

- `planGroup`: 기존 원본 도면 Plane과 벽·창문 Mesh를 소유한다.
- `finishGroup`: 실내 우드 바닥과 건물 외부 경계 표현을 소유하며 가구 배치 단계에서만 보인다.
- `furnitureGroup`: 사용자가 확정한 GLB 가구 Mesh를 소유하며 가구 배치 단계에서만 보인다.
- `placementPreviewGroup`: 현재 선택한 가구의 반투명 미리보기를 소유한다.
- 기존 전역 `floor`: 넓은 실외 지면으로 유지하되 재질만 절차적 아스팔트 재질로 교체한다.

단계 전환은 Mesh를 재생성하지 않고 각 그룹과 `isInputImage` Plane의 `visible` 값을 변경한다.

## 실내와 실외 바닥 판정

실내 영역은 원본 도면 색상을 사용하지 않고 인식 결과로 계산한다.

1. 도면 좌표계의 오프스크린 마스크에 벽 폴리곤을 장애물로 채운다.
2. 문과 창문 폴리곤도 실내 판정 단계에서만 임시로 채워 벽의 개구부를 막는다.
3. 캔버스 가장자리에서 flood fill을 수행해 외부 영역을 찾는다.
4. 장애물도 외부도 아닌 픽셀을 실내 영역으로 결정한다.
5. 이 실내 마스크를 alpha 채널로 사용한 CanvasTexture를 원본 도면과 같은 좌표 변환으로 배치한다.

문을 임시로 막는 처리는 바닥 판정에만 사용하며 실제 문 개구부와 벽 형상은 바꾸지 않는다. 내부 영역을 찾지 못하면 가구 배치 단계 진입을 막지 않고 도로 바닥만 표시하면서 상태 영역에 실내 바닥 판정 실패를 알린다.

## 재질과 조명

### 실내

- 밝은 오크색 바탕과 얇은 줄눈을 가진 반복 CanvasTexture를 사용한다.
- 실내 마스크 외부는 완전히 투명하게 만들어 도로 재질을 덮지 않는다.
- 바닥은 벽 바닥보다 아주 조금 높은 위치에 두어 z-fighting을 방지한다.

### 실외

- 넓은 지면은 중간 밝기의 청회색 아스팔트 바탕과 미세한 노이즈를 사용한다.
- 과도한 차선이나 장식은 넣지 않고 건물 바깥이 도로 표면처럼 읽히는 수준으로 제한한다.

### 벽

- `ExtrudeGeometry`의 cap 그룹에는 검은색 재질을, side 그룹에는 흰색 재질을 적용한다.
- 아래쪽 cap도 검은색이지만 지면 아래에 가려져 보이지 않는다.
- 창문 유리와 문 개구부 로직은 그대로 유지한다.

### 배경과 조명

- 장면 배경은 기존 베이지 회색 대신 밝은 청회색을 사용한다.
- HemisphereLight의 하늘색과 바닥색을 새 배경과 조화시키고, 기존 그림자와 벽의 흰색 가독성은 유지한다.
- 태양, 잔디, 하늘 애니메이션 같은 별도 장식 기능은 추가하지 않는다.

## 가구 카탈로그와 배치

- 카탈로그는 `/floor-plan-3d/furniture-assets/manifest.json`의 `items` 배열을 기준으로 만든다.
- 카테고리 필터와 파일명 검색을 제공한다.
- 1,680개 DOM 항목을 한 번에 만들지 않고 최초 60개를 표시한 뒤 `더 보기`로 확장한다.
- 카드 선택 시에만 `GLTFLoader`로 GLB를 불러오고, 동일 모델은 메모리 캐시를 재사용한다.
- `sizeMm`을 미터로 변환해 모델의 실제 목표 크기를 정하고 GLB의 BoundingBox를 기준으로 정규화한다.
- Raycaster가 실내 바닥 Plane과 만나는 위치를 실내 마스크 좌표로 변환해 배치 가능 여부를 판단한다.
- 확정된 배치 정보는 `id`, `relativePath`, `position`, `rotationY`, `sizeMm`만 저장한다.

## 상태와 내보내기

- 뷰 상태는 `original`, `structure`, `furnishing` 세 단계로 관리한다.
- 가구 배치는 현재 페이지 상태에 유지하고 구조 확인과 가구 배치 사이의 전환에서는 삭제하지 않는다.
- 기존 JSON 내보내기와 RoomLog 완료 payload에 선택적 `furnitures` 배열을 추가한다.
- 가구 하나가 로딩에 실패하면 해당 미리보기만 취소하고 기존 구조와 이미 배치된 가구는 유지한다.

## UI 구성

- 왼쪽 기존 패널에는 현재 단계 표시와 `구조 확인`, `다음: 가구 배치` 전환 버튼을 둔다.
- 가구 배치 단계에서 오른쪽에 카탈로그 패널을 표시한다.
- 선택된 가구가 있으면 패널 하단에 `이동`, `90° 회전`, `삭제` 버튼을 표시한다.
- 모바일에서는 오른쪽 패널을 화면 하단의 접을 수 있는 패널로 바꾼다.
- 상태 영역은 가구 로딩, 배치 불가 위치, 실내 바닥 판정 실패를 짧은 문장으로 알린다.

## 테스트 전략

- 순수 JavaScript 모듈에서 실내 flood fill, 문·창문 임시 폐쇄, 좌표 변환, 가구 크기 정규화, 배치 직렬화를 테스트한다.
- Python 뷰어 셸 테스트에서 새 모듈 import, 단계 버튼, GLTFLoader, 장면 그룹, 벽 cap/side 재질 연결을 확인한다.
- 기존 JavaScript와 Python 테스트를 모두 실행해 벽, 문, 창문, 상승 애니메이션 회귀가 없는지 확인한다.
- 실제 `http://localhost:3000/floor-plan-3d/mitunet`에서 도면 업로드부터 3D 구조 확인, 가구 배치 전환, GLB 로드, 배치·회전·삭제, 구조 확인 복귀를 브라우저로 검증한다.

## 완료 조건

- 가구 배치 단계에서 원본 도면 이미지가 보이지 않는다.
- 실내 우드 바닥과 실외 도로 바닥이 벽 경계를 기준으로 구분된다.
- 벽 측면은 흰색이고 상단 면은 검은색이다.
- 배경이 밝은 청회색으로 표시된다.
- 1,680개 가구 manifest를 불러오고 카테고리와 검색으로 항목을 찾을 수 있다.
- 가구를 실내에 배치하고 이동·회전·삭제할 수 있다.
- 구조 확인으로 돌아갔다가 다시 진입해도 배치 상태가 유지된다.
- 기존 벽 형상과 상승 애니메이션 테스트가 그대로 통과한다.
