# 3D 가구 플로팅 배치 도구 설계

## 목표

가구를 조작할 때 오른쪽 패널과 3D 화면을 번갈아 보지 않아도 되도록, 선택한 가구 바로 위에 아이콘형 조작 도구를 표시한다.

가구 이동 또는 신규 배치를 시작한 뒤에는 가구가 포인터를 계속 따라다니며, 사용자가 초록색 `배치 완료` 아이콘을 직접 누르기 전까지 위치를 확정하지 않는다.

## 변경 범위

- `viewer/index.html`의 가구 선택·이동·회전·삭제 UI와 이벤트 흐름
- 필요하면 화면 좌표 계산을 위한 작은 순수 JavaScript 보조 모듈
- 플로팅 도구 상태와 연결을 검증하는 JavaScript/Python 테스트

벽 생성, 창문·문 생성, 바닥 마스크, 재질, 카메라 전환, 3D 상승 애니메이션, GLB 크기 보정과 저장 데이터 형식은 변경하지 않는다.

## 사용자 흐름

### 1. 가구 선택 상태

3D에 이미 배치된 가구를 선택하면 가구 바로 위에 작은 아이콘 도구가 나타난다.

왼쪽부터 다음 네 버튼을 둔다.

1. 이동
2. 왼쪽으로 90도 회전
3. 오른쪽으로 90도 회전
4. 삭제

버튼에는 화면에 글자를 표시하지 않는다. 대신 마우스를 올렸을 때 의미를 확인할 수 있도록 `title`을 넣고, 스크린 리더를 위해 동일한 `aria-label`을 넣는다.

### 2. 기존 가구 이동 상태

`이동` 아이콘을 누르면 선택한 가구가 포인터를 따라다니기 시작한다. 이때 기존 위치와 회전값을 임시로 보관한다.

도구는 다음 두 아이콘으로 바뀐다.

- 빨간색 `×`: 이동 취소. 가구를 이동 전 위치와 회전값으로 되돌린다.
- 초록색 `✓`: 배치 완료. 현재 유효한 위치를 확정하고 포인터 추적을 끝낸다.

3D 바닥을 클릭하는 것만으로는 위치를 확정하지 않는다. 반드시 초록색 `✓`를 눌러야 한다.

### 3. 신규 가구 배치 상태

카탈로그에서 가구를 고르면 반투명 미리보기가 포인터를 따라다닌다. 기존 가구 이동과 같은 `×`/`✓` 도구를 표시한다.

- `×`를 누르면 신규 미리보기를 제거한다.
- `✓`를 누르면 현재 유효한 위치에 새 가구를 확정한다.
- 실내가 아니거나 다른 가구와 겹쳐 배치 불가능한 위치에서는 `✓`를 비활성화한다.

배치가 완료되면 새 가구는 선택된 상태로 남고, 도구는 네 개의 일반 조작 아이콘으로 돌아간다.

## 플로팅 도구 위치

- HTML 오버레이를 Three.js 캔버스가 들어 있는 영역 위에 겹쳐 표시한다.
- 선택 또는 배치 중인 가구의 `THREE.Box3`를 계산하고, 상단 중앙 월드 좌표를 카메라로 화면 좌표에 투영한다.
- 가구와 겹치지 않도록 화면상으로 작은 여백을 두고 위에 표시한다.
- 카메라 이동·회전·줌, 창 크기 변경, 가구 이동 중에는 위치를 다시 계산한다.
- 화면 가장자리에서는 도구가 잘리지 않도록 좌우와 위아래를 캔버스 범위 안으로 제한한다.
- 대상 가구가 없거나 가구가 카메라 뒤에 있거나 가구 배치 화면이 아니면 숨긴다.

## 상태와 기존 코드 연결

기존 상태를 그대로 단일 기준으로 사용한다.

- `selectedFurniture`: 확정되어 선택된 가구
- `pendingFurniture`: 신규 배치 또는 기존 가구 이동 중인 미확정 가구

기존 함수도 가능한 한 재사용한다.

- 이동 시작: `beginMoveSelectedFurniture()`
- 배치 확정: `confirmFurniturePlacement()`
- 취소와 원상 복구: `cancelFurnitureInteraction()`
- 삭제: `deleteSelectedFurniture()`
- 회전: `rotateSelectedFurniture(delta)` 형태로 바꾸어 `-90도`, `+90도`를 모두 지원

`updateFurnitureInteractionUi()`는 현재 상태에 따라 도구를 숨기거나, 네 개의 선택 도구 또는 두 개의 배치 도구 중 하나를 표시한다.

기존 오른쪽 패널의 `이동`, `90도 회전`, `삭제`, `배치 취소` 버튼은 플로팅 도구와 기능이 중복되므로 제거한다. 카탈로그, 검색, 페이지 이동과 `전체 초기화` 기능은 유지한다.

## 입력 충돌 방지

- 플로팅 도구의 `pointerdown`과 `click` 이벤트는 전파를 막아 캔버스 선택과 OrbitControls가 동시에 반응하지 않게 한다.
- 배치 중 포인터가 도구 위로 이동하면 가구는 마지막으로 계산된 위치에서 잠시 멈춘다. 따라서 흔들리지 않는 상태에서 `×` 또는 `✓`를 누를 수 있다.
- 도구 바깥으로 포인터가 돌아가면 가구가 다시 포인터를 따라다닌다.
- 기존 `Escape` 취소 동작은 유지한다.

## 회전과 유효성

- 왼쪽 회전은 `-90도`, 오른쪽 회전은 `+90도`씩 적용한다.
- 회전 후 가구 바닥 면적이 실내 마스크를 벗어나거나 다른 가구와 겹치면 회전을 적용하지 않고 기존 회전값을 유지한다.
- 미확정 가구가 유효하지 않은 위치에 있으면 기존의 오류 색상과 안내 문구를 유지하고 `✓`를 누를 수 없게 한다.

## 접근성과 화면 크기

- 각 아이콘 버튼의 실제 클릭 영역은 최소 `40px × 40px`로 만든다.
- 키보드 포커스가 보이도록 `:focus-visible` 스타일을 제공한다.
- 좁은 화면에서도 도구가 캔버스 밖으로 나가지 않도록 위치를 제한한다.
- 색상만으로 의미를 구분하지 않도록 `×`, `✓`의 모양과 `aria-label`을 함께 사용한다.

## 테스트

### 자동 테스트

- 선택 없음, 선택됨, 배치 중 상태별로 올바른 도구가 표시되는지 검증
- 신규 배치 취소 시 미리보기가 제거되는지 검증
- 기존 가구 이동 취소 시 원래 위치와 회전으로 복구되는지 검증
- 유효하지 않은 위치에서 `✓`가 비활성화되는지 검증
- 왼쪽·오른쪽 회전이 각각 정확히 90도이고 실패 시 원래 회전을 유지하는지 검증
- 캔버스 클릭만으로 배치가 확정되지 않는지 검증
- 화면 좌표 투영과 가장자리 제한 계산을 순수 함수로 분리할 경우 해당 계산 검증
- 기존 벽·문·창문·바닥·상승 애니메이션 관련 테스트가 그대로 통과하는지 검증

### 브라우저 확인

`http://localhost:3000/floor-plan-3d/mitunet`에서 다음 흐름을 확인한다.

1. 도면 업로드 후 3D 렌더링과 가구 배치 화면 진입
2. 기존 가구 선택 시 네 아이콘이 가구 위에 나타나는지 확인
3. 카메라 회전·줌 중 도구가 가구를 따라가는지 확인
4. 이동 시작 후 바닥 클릭만으로 확정되지 않고 계속 포인터를 따라가는지 확인
5. `✓`로 확정되고 `×`로 기존 위치가 복구되는지 확인
6. 신규 가구의 `×` 취소와 `✓` 확정 확인
7. 좌우 90도 회전과 삭제 확인
8. 벽·문·창문·바닥과 상승 애니메이션이 이전과 동일한지 확인

## 완료 조건

- 선택한 가구 바로 위에서 이동·좌회전·우회전·삭제를 아이콘으로 실행할 수 있다.
- 이동 또는 신규 배치 중에는 `×`와 `✓`만 보인다.
- `✓`를 누르기 전까지 가구가 포인터를 따라다니고 위치가 확정되지 않는다.
- `×`를 누르면 기존 가구는 원상 복구되고 신규 가구는 제거된다.
- 배치 불가능한 위치에서는 `✓`가 비활성화된다.
- 오른쪽 패널의 중복 조작 버튼은 사라지고 카탈로그와 전체 초기화는 유지된다.
- 벽을 포함한 가구 외 3D 생성 로직과 애니메이션에는 변경이 없다.
