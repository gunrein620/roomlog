# 자유 축척 점 선택 설계

## 목표

2D 검토 화면의 `Scale` 도구에서 벽 꼭짓점 근처만 선택할 수 있는 제한을 없애고, 도면 영역 안에서 사용자가 클릭한 정확한 위치를 축척 측정점으로 사용한다.

## 확정 동작

- 첫 번째 클릭은 도면 내부의 정확한 이미지 좌표를 첫 측정점으로 저장한다.
- 두 번째 클릭은 도면 내부의 정확한 이미지 좌표를 둘째 측정점으로 저장한다.
- 같은 좌표를 연속으로 선택하면 0 길이 보정을 막기 위해 둘째 클릭을 무시한다.
- 두 점이 이미 선택된 상태에서 다시 클릭하면 기존 보정을 지우고 해당 위치를 새 측정의 첫 점으로 사용한다.
- 도면 영역 밖에서 시작된 클릭은 기존처럼 무시한다.
- 선택점 원과 두 점 사이의 선, 실제 길이 입력, `mm/px` 계산, 벽 길이 표시는 기존 동작을 유지한다.

## 코드 변경 범위

### 2D 편집기

`viewer/review-editor.mjs`의 축척 도구 포인터 처리에서 꼭짓점 탐색과 스냅 반경 검사를 제거한다. 이미 도면 내부로 검증되고 경계 안으로 정규화된 `point`를 그대로 `scalePoints`에 저장한다.

꼭짓점 스냅만을 위해 존재하는 다음 코드는 함께 제거한다.

- `SCALE_SNAP_RADIUS_PX`
- `extractWallVertices()`
- `nearestVertex()`
- `wallVertices` 캐시와 로딩 시 폴리곤 꼭짓점 추출

축척 적용 전 오류 문구도 `두 벽 꼭짓점을 선택`에서 `두 점을 선택`으로 바꾼다.

### 화면 안내 문구

`viewer/index.html`의 다음 안내에서 `wall corner` 표현을 제거하고 자유 선택을 명확히 한다.

- 초기 안내: Scale 도구를 선택하고 도면에서 두 점을 클릭
- 첫 점 선택 후 안내: 둘째 점을 클릭
- 기본 축척 요약: 두 점 선택

## 데이터 흐름

1. 포인터의 화면 좌표를 이미지 좌표로 변환한다.
2. 이미지 영역 안인지 검사한다.
3. 경계 안으로 정규화된 정확한 좌표를 `scalePoints`에 저장한다.
4. 두 점과 사용자가 입력한 실제 길이로 기존 `calibrationFromMeasurement()`를 호출한다.
5. 기존 벽면 치수 캐시를 재계산하고 화면을 다시 그린다.

## 테스트

- 벽 꼭짓점 목록이 비어 있어도 도면 내부 클릭 좌표가 그대로 첫 점에 저장되는지 확인한다.
- 둘째 점도 정확한 클릭 좌표로 저장되는지 확인한다.
- 동일 좌표를 두 번 클릭하면 둘째 점이 추가되지 않는지 확인한다.
- 두 점 이후 클릭하면 새 첫 점으로 초기화되는지 확인한다.
- 도면 밖 클릭은 저장되지 않는 기존 동작을 유지한다.
- 화면 안내 문구에 `wall corner`가 남아 있지 않은지 확인한다.
- 전체 JavaScript 및 관련 뷰어 테스트를 다시 실행한다.

## 변경하지 않는 부분

- 원본 벽 마스크와 벽 인식 결과
- 문·창문 인식 및 치수 연속 처리
- 벽 길이 계산 방식
- 3D 벽 생성 및 렌더링
- 저장·전송되는 축척 데이터 형식
