### 0. 공통 정책 및 안전장치

- 계약서, 사진, 계좌, 주소, 호수, 얼굴, 우편물 등 민감정보는 필요 시 마스킹 또는 삭제할 수 있어야 한다.
- 계약서 OCR, 사진 저장, 업체 전달, 번역, 기록 보관에 대해서는 사용자 동의를 받는다.
- 임차인, 임대인/관리자, 협력업체는 접근 가능한 정보 범위가 다르다.
- 임차인은 본인 계약, 본인 민원, 본인 사진 기록만 조회할 수 있다.
- 협력업체에는 수리에 필요한 사진, 증상 요약, 방문 가능 시간, 대략적 위치만 전달한다.
- 계약서 전문, 상세 주소, 연락처 등은 필요한 경우에만 단계적으로 제공한다.
- AI는 책임 소재를 확정하지 않고, `임대인 책임 가능성`, `임차인 책임 가능성`, `판단 어려움`으로만 표시한다.
- 임차인과 임대인은 AI 판단 결과에 대해 수정 요청 또는 이의 제기를 할 수 있다.
- 수정된 긴급도, 책임 가능성, 하자 유형은 이후 모델 개선 및 기록 정정 데이터로 저장한다.
- 계약 종료 후 임차인은 계약서 OCR, 사진, 민원 기록 삭제를 요청할 수 있다.
- 단, 정산 또는 분쟁 확인에 필요한 기록은 별도 보관 기간을 둘 수 있다.
- 보증금 반환 금액은 확정값이 아니라 예상/참고용으로 표시한다.
- 퇴실 리포트는 보증금 차감 판단이 아니라, 퇴실 정산에 참고할 수 있는 증빙 기록 정리로 표현한다.
- 임차인 화면과 임대인/관리자 화면은 같은 원천 데이터를 역할별로 다르게 보여준다.
- 민원/하자 티켓, 사진, 채팅, 상태 변경, 답변 기록은 하나의 타임라인에 연결해 관리한다.

### 1. 계약서 사진 업로드 및 OCR 분석

- 임차인이 계약서 사진 또는 PDF를 업로드하면 OCR로 텍스트화한다.
- 계약서 작성 전 주의해야 할 조항을 AI가 분석해 안내한다.
- 보증금, 월세, 관리비, 계약 기간, 특약, 수리 책임, 원상복구, 청소비 관련 조항을 추출한다.
- 계약 후에는 계약서 원본, OCR 전문을 DB에 저장한다.
- 이후 민원/하자 발생 시 계약서 내용을 조회하여 참고 근거로 활용한다.
- 임대인/관리자 화면에서도 같은 계약서 기록을 검토할 수 있다.
- AI 분석 결과는 확정 판단이 아니라 참고 정보로 제공한다.

예시 표시:

```
책임 가능성: 임대인 확인 필요
근거 후보: 계약서 내 에어컨이 기본 옵션으로 명시되어 있음
주의: 최종 판단은 임대인 확인이 필요합니다.
```

### 2. 입주 전 체크리스트 및 이미지 아카이빙 - 보류

- 임차인이 입주 전 방 상태를 자세히 기록할 수 있도록 체크리스트를 제공한다.
- 계약서에 적힌 옵션을 참고해 확인해야 할 항목을 자동으로 제안한다.
- 화장실, 싱크대, 벽지, 바닥, 창문, 도어락, 에어컨, 세탁기 등 항목별 사진을 저장한다.
- 계약서 내용과 다른 추가 사항도 사진과 메모로 기록할 수 있다.
- 입주 전/후 사진 비교를 위해 같은 위치에서 촬영할 수 있도록 촬영 가이드를 제공한다.
- 사진은 매물별, 공간별, 항목별, 날짜별로 저장한다.
- 저장된 사진은 이후 민원/하자/퇴실 리포트에서 활용한다.
- 임대인/관리자는 담당 호실 기준으로 해당 사진 기록을 확인할 수 있다.

촬영 가이드 예시:

```
- 정면에서 촬영해주세요.
- 같은 공간은 전체 사진 1장, 문제 부위 확대 사진 1장을 남겨주세요.
- 퇴실 비교를 위해 문, 창문, 가전, 벽면이 함께 보이게 찍어주세요.
```

### 3. 민원/하자 접수 리얼타임 챗봇 중요도 높음 / 의존성 높음

- 임차인은 채팅 또는 음성으로 민원/하자 내용을 접수할 수 있다.
- 음성으로 말한 내용은 실시간으로 텍스트로 변환되어 채팅창에 함께 표시된다.
- 임차인은 음성 대화 중에도 채팅창에 추가 설명을 입력할 수 있다.
- AI는 임차인의 음성, 채팅 메시지, 사진을 함께 참고해 민원 접수 초안을 생성한다.
- AI가 사진이 필요하다고 판단하면 “문제 부위 사진을 보내주세요”라고 요청한다.
- 임차인은 채팅창에서 사진을 첨부할 수 있다.
- 첨부된 사진은 민원 내용과 함께 분석되어 문제 유형, 긴급도, 추가 확인 필요 항목을 정리한다.
- 입력 내용을 분석해 민원 유형을 자동 분류한다.
- 분류 기준은 `하자`, `소음`, `설비`, `납부`, `계약`, `공용공간`, `기타`로 나눈다.
- 긴급도는 `1순위 긴급`, `2순위 우선 처리`, `3순위 일반 수리`, `4순위 일반 문의`로 구분한다.
- 위험 키워드가 포함된 경우 긴급도를 강제로 상향한다.
- 접수 전 임차인에게 AI가 정리한 요약 내용을 보여주고 수정할 수 있게 한다.
- 임차인이 최종 확인한 내용만 민원으로 접수한다.
- 접수된 민원은 서버에서 하나의 티켓으로 생성된다.
- 같은 티켓이 임차인 화면과 임대인/관리자 화면에 역할별로 다르게 표시된다.
- 임차인 화면에는 본인이 입력한 내용, 처리 상태, 요청받은 추가 정보, 임대인 답변이 표시된다.
- 임대인/관리자 화면에는 AI 요약, 긴급도, 사진 분석, 책임 가능성, 처리 이력이 표시된다.
- AI는 책임 소재를 확정하지 않고 참고 정보만 제공한다.
- 접수 후 임차인은 민원 처리 상태를 확인할 수 있다.
- 상태값은 `접수됨`, `검토중`, `추가정보 요청`, `업체 배정`, `수리중`, `완료`, `재요청`으로 관리한다.
- 콜봇처럼 전화망을 사용하는 기능은 MVP 이후 확장 기능으로 분리하고, 초기에는 앱/웹 기반 음성 챗봇을 우선 적용한다.

입력 항목 예시:

```
- 민원 내용
- 발생 위치
- 발생 시점
- 음성 대화 기록
- 채팅 메시지
- 사진 첨부
- 반복 발생 여부
- 방문 가능 시간
```

접수 요약 예시:

```
민원 유형: 하자
세부 유형: 누수
긴급도: 1순위 긴급
발생 위치: 305호 화장실 천장
사진 여부: 첨부됨
추가 확인 필요: 누수가 계속 진행 중인지 확인 필요
추천 조치: 임대인 즉시 확인 필요
안내: 최종 확인 및 조치는 임대인 확인 후 진행됩니다.
```

처리 흐름:

```
임차인 음성/채팅 입력
→ AI가 민원 내용 실시간 정리
→ 필요한 경우 사진 첨부 요청
→ 사진 분석 및 추가 질문 생성
→ 접수 초안 생성
→ 임차인 확인/수정
→ 민원 접수 완료
→ 같은 티켓이 임대인/관리자 화면에 전달
→ 처리 상태 확인
```

### 4. 하자 사진 분석

- 임차인이 하자 사진을 업로드하면 AI가 사진 속 문제 후보를 탐지한다.
- 예: 누수, 곰팡이, 벽지 훼손, 바닥 손상, 에어컨 문제, 창틀 문제 등.
- AI는 사진 분석 결과와 텍스트 민원 내용을 함께 종합한다.
- 이전 체크리스트 사진과 비교해 기존 하자인지, 새로 발생한 문제인지 참고 정보를 제공한다.
- 사진 각도, 조명, 해상도, 촬영 위치가 크게 다르면 `비교 어려움`으로 표시하고 재촬영을 요청한다.
- 임대인/관리자가 추가 사진을 요청하면 임차인 화면에 요청 항목이 표시된다.
- 임차인이 추가 사진을 제출하면 기존 티켓에 자동 연결된다.
- 추가 정보가 들어오면 AI 요약, 긴급도, 사진 분석 결과를 부분 갱신할 수 있다.
- 책임 소재는 확정하지 않고 가능성으로만 표시한다.

표시 예시:

```
문제 후보: 에어컨 실내기 물샘
책임 가능성: 임대인 확인 필요
근거 후보:
- 입주 전 사진에 동일 증상 없음
- 계약서에 에어컨이 기본 옵션으로 기재됨
- 설비 자연 고장 가능성 있음
주의: 최종 판단은 임대인 확인 후 확정됩니다.
```

### 5. 긴급 민원/하자 판단

- 민원/하자 접수 시 AI가 유형과 긴급도를 함께 판단한다.
- 긴급도는 `1순위 긴급`, `2순위 우선 처리`, `3순위 일반 수리`, `4순위 일반 문의`로 구분한다.
- 긴급도는 임대인/관리자 대시보드와 임차인 화면에 함께 표시된다.
- 위험 키워드가 포함되면 AI 판단과 별개로 긴급도가 자동 상향된다.
- 긴급도에 따라 임대인 알림, 업체 연결, 추가 사진 요청 여부가 달라진다.
- 긴급도 판단 결과에는 판단 이유와 추천 조치를 함께 표시한다.
- 임차인은 AI가 판단한 긴급도에 대해 이의제기하거나 추가 설명을 남길 수 있다.
- 임대인/관리자가 최종 수정한 긴급도는 같은 티켓에 반영된다.

| 순위 | 이름 | 처리 기준 | 핵심 판단 기준 |
| --- | --- | --- | --- |
| 1순위 | 긴급 | 즉시 확인 / 당일 대응 | 안전 위험, 필수 기능 중단, 피해 확산 가능 |
| 2순위 | 우선 처리 | 빠른 확인 / 1~2일 내 처리 | 방치하면 악화될 수 있거나 생활 불편이 큰 문제 |
| 3순위 | 일반 수리 | 일정 조율 / 3~7일 내 처리 | 수리는 필요하지만 즉시 위험이나 확산 가능성은 낮음 |
| 4순위 | 일반 문의 | 확인 후 답변 / 예약 처리 | 단순 문의, 경미한 불편, 기록성 요청 |

위험 키워드 예시:

```
가스 냄새
불꽃
누전
물이 계속 샘
천장에서 물
보일러 완전 고장
수도 안 나옴
문이 안 잠김
침수
화재
```

### 6. 임차인 책임 가능성이 높은 하자 처리

- AI가 계약서, 입주 전 사진, 현재 사진, 민원 내용을 참고해 임차인 책임 가능성이 높은 하자로 판단하면 임차인에게 안내한다.
- 단, 확정 표현은 사용하지 않고 `임차인 부담 가능성이 있음` 또는 `임차인 확인 필요`로 표시한다.
- 임차인은 책임 가능성 판단에 대해 추가 설명을 남기거나 이의제기할 수 있다.
- 긴급도가 1~2순위인 경우 업체 연결을 우선 제안한다.
- 긴급도가 3순위인 경우 일정 조율 가능한 일반 수리로 안내한다.
- 업체 연결 전 임차인은 전달될 정보 범위를 확인하고 동의한다.
- 전달 정보는 하자 사진, 증상 요약, 방문 가능 시간, 대략적 위치로 제한한다.
- 계약서 전문, 상세 주소, 연락처는 필요한 경우에만 단계적으로 제공한다.
- 업체 견적 확인, 일정 조율, 결제 주체, 취소, 환불, 수리 완료 확인 기능을 포함한다.

처리 흐름:

```
하자 접수
→ 긴급도 판단
→ 임차인 책임 가능성 표시
→ 임차인 확인 또는 이의제기
→ 업체 추천 여부 안내
→ 임차인 동의
→ 사진/증상 요약 전달
→ 견적 확인
→ 일정 조율
→ 직접 결제
→ 수리 완료 기록 저장
```

### 7. 임대인/관리자 책임 가능성이 높은 긴급 하자 처리

- AI가 임대인 또는 관리자 책임 가능성이 높고 긴급도가 높은 문제로 판단하면 임대인/관리자에게 즉시 전달한다.
- `1순위 긴급` 민원은 임대인/관리자 대시보드 상단에 표시하고 별도 알림을 보낸다.
- `2순위 우선 처리` 민원은 빠른 확인 대상 카드로 분류한다.
- 임대인/관리자가 미리 설정한 경우, 협력업체 출동 요청을 자동 또는 반자동으로 진행할 수 있다.
- 자동 출동 여부는 임대인/관리자가 `on/off`로 설정할 수 있다.
- 인앱 포인트 또는 예치금 차감은 임대인/관리자 승인 후 진행한다.
- 협력업체 파견이 어려운 경우 문제 사진과 요약을 업체들에게 전달해 빠른 견적 또는 출동 가능 여부를 확인한다.
- 최종 출동, 견적 승인, 결제 주체 확정은 임대인/관리자 설정과 승인 단계에 따라 처리된다.
- 임차인은 업체 배정, 방문 일정, 수리 진행 상태를 같은 티켓에서 확인한다.

처리 흐름:

```
하자 접수
→ 긴급도 1~2순위 판단
→ 임대인/관리자 책임 가능성 표시
→ 임대인/관리자 즉시 알림
→ 임대인/관리자 확인
→ 협력업체 출동 요청
→ 견적 승인
→ 포인트 차감 또는 별도 결제
→ 수리 완료 기록 저장
→ 임차인 화면에 완료 상태 반영
```

### 8. 공지/메시지/민원 번역

- 임대인/관리자 측에서 발송하는 공지, 메시지, 민원 처리 내역, 고지서 등을 외국인 임차인에게 맞게 번역한다.
- 번역문만 보여주지 않고 원문도 함께 표시한다.
- 임차인이 외국어로 작성한 민원/하자 내용도 한국어로 번역되어 임대인/관리자에게 전달된다.
- 계약서, 고지서, 정산 안내처럼 문맥이 중요한 문서는 단순 번역 API보다 용어 사전 또는 계약 문맥 기반 AI 번역을 검토한다.
- 번역 오류를 방지하기 위해 모든 번역 메시지는 원문과 번역문을 함께 저장한다.
- 분쟁 가능성이 있는 계약서, 고지서, 정산 안내는 원문을 기준으로 한다.

표시 예시:

```
원문: 내일 오전 10시부터 수도 점검이 있습니다.
번역: Water inspection will take place from 10 AM tomorrow.
```

### 9. 민원/하자 수리 기록 조회

- 임차인은 본인이 접수한 민원과 하자 처리 내역을 기간별로 조회할 수 있다.
- 1개월, 3개월, 6개월 등 기간을 선택해 과거 기록을 모아볼 수 있다.
- 임차인이 보는 기록은 임대인/관리자 화면의 같은 티켓 데이터를 기반으로 한다.
- 임대인/관리자 내부 상태와 임차인에게 보여줄 상태 문구는 다르게 표시할 수 있다.
- 상태값은 `접수됨`, `검토중`, `추가정보 요청`, `업체 배정`, `수리중`, `완료`, `재요청`으로 표시한다.
- 동일 유형 민원이 3개월 내 2회 이상 발생하면 반복 민원으로 감지한다.
- 반복 민원은 임대인/관리자 대시보드에도 표시해 개선 필요 이슈로 제안한다.
- 민원별 사진, 채팅, 임대인 답변, 수리 완료 여부를 함께 확인할 수 있다.
- 추가 사진 요청, 답변 전송, 업체 배정, 완료 확인 내역은 티켓 타임라인에 저장된다.

상태 표시 매핑 예시:

| 임대인/관리자 상태 | 임차인 표시 |
| --- | --- |
| 접수 | 접수됨 |
| 검토 | 검토중 |
| 추가 정보 요청 | 추가정보 요청 |
| 업체 배정 | 업체 배정 |
| 견적 확인 | 처리 준비중 |
| 수리 중 | 수리중 |
| 완료 | 완료 |
| 재요청 | 재요청 |

### 10. 퇴실 대비 기록 리포트 생성

- 입주 전 사진, 하자 접수, 수리 기록, 채팅 기록을 누적 저장한다.
- 퇴실 전 AI가 참고용 기록 리포트를 생성한다.
- 리포트에는 훼손 추정 항목, 기존 기록 여부, 수리 이력, 관련 사진, 채팅 근거가 포함된다.
- 기존 옵션 존재 여부를 다시 체크할 수 있는 퇴실 체크리스트를 제공한다.
- 계약서에 따른 원상복구, 청소비, 추가 비용 발생 가능 항목을 참고용으로 안내한다.
- 보증금 차감 여부를 판단하지 않고, 보증금 정산 전 참고할 증빙 기록으로 제공한다.
- 임대인/관리자 화면의 호실 기록 리포트와 같은 원천 데이터를 사용한다.
- 임차인 화면에는 임차인이 확인 가능한 범위의 기록만 표시한다.
- 리포트에는 `참고자료이며 최종 정산은 임대인 확인 후 확정됩니다` 문구를 함께 표시한다.

### 11. 이번 달 청구·납부·연체 확인

- 임차인은 이번 달 월세, 관리비, 납부 기한을 확인할 수 있다.
- 납부 계좌번호, 예금주, 납부 상태를 표시한다.
- 납부 예정, 납부 완료, 연체 상태를 구분해 보여준다.
- 납부 기한 전 알림과 연체 알림을 제공한다.
- 연체 발생 시 금액, 기준일, 안내 메시지를 보여주고 임대인 문의로 연결할 수 있다.
- 임대인/관리자가 입력하거나 업로드한 청구 데이터를 기반으로 임차인 화면에 표시한다.
- 임대인/관리자가 납입 안내문이나 독촉문을 발송한 경우 발송 이력은 청구 기록에 연결된다.
- 자동 계좌 연동은 후순위로 두고, 초기에는 임대인/관리자 입력 또는 파일 업로드 기반으로 처리할 수 있다.

### 12. 퇴실 정산 전 정보 확인

- 계약서상 계약 종료일까지 남은 기간을 표시한다.
- 보증금, 미납 월세, 미납 관리비, 수리비 후보 항목을 바탕으로 예상 정산 정보를 제공한다.
- 입주 전 사진, 하자 기록, 수리 기록, 퇴실 체크리스트를 함께 참고한다.
- 보증금 반환 금액은 확정값이 아니라 참고용 예상 금액으로 표시한다.
- 최종 정산은 임대인 확인 후 확정된다는 문구를 함께 제공한다.
- 임차인은 예상 정산 항목에 대해 문의하거나 이의제기할 수 있다.
- 임대인/관리자가 확정한 정산 결과는 별도 상태로 표시한다.

표시 예시:

```
계약 종료까지 남은 기간: 24일
예상 확인 항목:
- 미납 관리비 여부
- 원상복구 확인 필요 항목
- 수리 완료 여부
주의: 최종 정산 금액은 임대인 확인 후 확정됩니다.
```

### 13. 관리비 사용 내역 확인

- 임차인은 관리비 청구 내역과 사용 항목을 확인할 수 있다.
- 전기, 수도, 청소, 공용관리, 수선비 등 항목별 사용 내역을 표시한다.
- 임대인/관리자가 등록한 영수증 또는 비용 기록이 있는 경우 함께 확인할 수 있다.
- 관리비 투명성을 높이고 임차인의 불필요한 문의를 줄이는 기능으로 활용한다.
- 임차인은 관리비 사용 내역에 대해 문의를 남길 수 있다.
- 임대인/관리자가 데이터를 입력하지 않으면 기능 가치가 낮기 때문에 임대인/관리자 측 입력 플로우가 필요하다.

### 14. 알림 및 커뮤니케이션

- 민원 접수, 추가 정보 요청, 임대인 확인, 업체 배정, 방문 일정 확정, 수리 완료 시 임차인에게 알림을 제공한다.
- 임차인은 민원별 채팅 또는 댓글 형태로 추가 설명을 남길 수 있다.
- 임대인/관리자가 추가 사진이나 추가 설명을 요청하면 임차인 화면에 요청 항목이 표시된다.
- 임차인이 추가 사진 또는 설명을 제출하면 기존 티켓에 자동 연결된다.
- 추가 정보가 들어오면 AI 요약, 긴급도, 사진 분석 결과를 부분 갱신할 수 있다.
- 모든 대화와 알림 기록은 해당 민원/하자 기록에 연결되어 추후 조회할 수 있다.
- 임대인/관리자 답변, 상태 변경, 업체 배정, 수리 완료 내역은 임차인 화면에 역할별로 표시된다.
- 외국인 임차인의 경우 원문과 번역문을 함께 저장한다.
- 알림은 앱 알림, 문자, 카카오 알림톡 등으로 확장 가능하나 MVP에서는 앱 내 알림 중심으로 시작한다.

### 15. 권한 및 데이터 관리

- 임차인은 본인 계약과 본인 민원 기록만 조회할 수 있다.
- 임대인/관리자는 담당 건물/호실의 기록만 조회할 수 있다.
- 협력업체는 수리에 필요한 최소 정보만 확인할 수 있다.
- 계약서, 사진, 계좌, 주소 등 민감정보는 마스킹 또는 접근 제한을 적용한다.
- 업체 전달 전 임차인 또는 임대인/관리자는 전달될 정보 범위를 확인하고 동의한다.
- 전달 정보는 하자 사진, 증상 요약, 방문 가능 시간, 대략적 위치로 제한한다.
- 계약서 전문, 상세 주소, 연락처는 필요한 경우에만 단계적으로 제공한다.
- 계약 종료 후 기록 삭제 요청 및 보관 기간 정책을 제공한다.
- AI 판단 결과, 임대인/관리자 수정값, 임차인 이의제기 내역은 변경 이력으로 저장한다.
- 기록 수정 시 누가, 언제, 어떤 항목을 변경했는지 로그로 남긴다.

### 16. 민원 처리 상태, 알림, 확인 흐름 보강

- 민원/하자 티켓은 임차인이 현재 상황을 쉽게 이해할 수 있도록 임차인용 상태 문구와 다음 행동을 함께 표시한다.
- 상태 변경이 발생하면 해당 티켓의 채팅/알림/타임라인에 동시에 기록한다.
- 임차인에게는 내부 처리 상태를 그대로 노출하지 않고, 이해하기 쉬운 문구로 변환해 보여준다.
- 긴급도 1순위 민원은 접수 직후 임대인/관리자 확인 대기 상태와 예상 다음 조치를 함께 안내한다.
- 추가 정보 요청 상태에서는 필요한 사진, 촬영 위치, 설명 항목을 구체적으로 제시한다.
- 업체 배정 상태에서는 업체명, 방문 예정 시간, 방문 전 확인 사항을 표시한다.
- 수리 완료 상태에서는 임차인이 완료 확인, 미해결 재요청, 추가 사진 등록 중 하나를 선택할 수 있다.
- 임차인이 일정 시간 내 완료 확인을 하지 않으면 자동 완료 처리 여부를 안내하되, 재요청 가능 기간을 함께 표시한다.
- 상태 알림은 앱 알림을 기본으로 하고, 긴급 또는 방문 일정 관련 알림은 문자/알림톡으로 확장할 수 있다.

상태별 임차인 행동 예시:

```
접수됨: 접수 요약 확인 / 내용 수정 요청 가능
검토중: 관리자 확인 대기 / 추가 설명 남기기 가능
추가정보 요청: 사진 또는 설명 제출
업체 배정: 방문 일정 확인 / 일정 조율
수리중: 진행 상태 확인 / 현장 방문 확인
완료: 완료 확인 / 재요청 또는 이의제기
재요청: 미해결 사유 입력 / 추가 사진 첨부
```

### 17. 이의제기, 정정, 중복 접수 처리

- 임차인은 AI 요약, 민원 유형, 긴급도, 책임 가능성, 완료 처리 결과에 대해 이의제기할 수 있다.
- 이의제기 시 임차인은 사유, 추가 설명, 사진, 원하는 조치 방향을 함께 남길 수 있다.
- 이의제기는 기존 티켓에 연결되며 별도 민원으로 분리하지 않는다.
- 관리자가 이의제기를 검토하면 검토 결과와 변경된 상태를 임차인 화면에 표시한다.
- AI 요약이 틀렸거나 음성 인식 결과가 부정확한 경우 임차인이 접수 전 또는 접수 후 정정 요청을 할 수 있다.
- 같은 호실, 같은 위치, 같은 유형의 민원이 짧은 기간 안에 반복 접수되면 중복 가능성을 안내한다.
- 중복 가능성이 있는 경우 임차인은 기존 티켓에 추가 설명을 붙일지, 새 티켓으로 접수할지 선택할 수 있다.
- 관리자가 중복 티켓을 병합하면 병합 이력을 임차인 화면에 남긴다.
- 민원이 취소되거나 잘못 접수된 경우 취소 사유를 저장하고, 이미 전달된 업체 요청이나 알림이 있으면 함께 정리한다.

이의제기 입력 예시:

```
이의제기 대상: 긴급도
임차인 의견: 물이 계속 떨어지고 있어 일반 수리가 아니라 긴급 확인이 필요합니다.
추가 사진: 첨부됨
요청 조치: 오늘 중 관리자 확인 요청
```

### 18. 협력업체 방문 및 개인정보 제공 동의

- 협력업체가 배정되는 경우 임차인에게 전달될 정보 범위를 먼저 보여준다.
- 기본 전달 정보는 하자 사진, 증상 요약, 방문 가능 시간, 대략적 위치, 티켓 번호로 제한한다.
- 상세 주소, 호수, 연락처는 방문 일정 확정 또는 관리자 승인 이후 필요한 범위에서만 제공한다.
- 임차인은 방문 가능 시간대를 선택하거나 변경 요청을 남길 수 있다.
- 방문 일정이 확정되면 임차인 화면에 업체명, 방문 예정 시간, 준비 사항을 표시한다.
- 방문자가 변경되거나 일정이 지연되는 경우 임차인에게 즉시 알림을 제공한다.
- 수리 완료 후 임차인은 완료 여부, 만족도, 미해결 사유, 추가 사진을 남길 수 있다.
- 임차인이 완료를 거절하면 티켓은 `재요청` 상태로 돌아가고 관리자 화면에 재검토 대상으로 표시된다.
- 협력업체와의 직접 채팅은 필요한 경우에만 열고, 모든 대화는 티켓 타임라인에 저장한다.

방문 안내 예시:

```
업체명: 가나다수리
방문 예정: 6월 30일 오후 2시~4시
전달 정보: 누수 사진 2장, 증상 요약, 방문 가능 시간
임차인 확인 필요: 방문 가능 여부 확인
주의: 상세 연락처는 방문 확정 후 필요한 경우에만 제공됩니다.
```

### 19. 수리비 안전결제 및 환불 흐름

- 임차인 부담 가능성이 있는 수리에서 견적을 승인하면 수리비를 바로 업체에 지급하지 않고 안전결제 또는 예치금 형태로 보관할 수 있다.
- 결제 전 임차인은 견적 금액, 결제 주체, 결제 수단, 환불 조건을 확인한다.
- 결제 주체는 `임차인 부담`, `임대인 부담`, `판단 대기`로 구분해 표시한다.
- 임차인 부담 수리비는 임차인이 결제하고, 수리 완료 확인 전까지 예치 상태로 관리한다.
- 수리 완료 후 임차인이 완료 확인을 하면 업체 지급 절차가 진행된다.
- 임차인이 미해결 또는 재요청을 선택하면 예치금 지급은 보류되고 관리자 재검토 대상으로 전환된다.
- 수리 취소, 업체 미방문, 분쟁 발생 시 환불 요청 또는 환불 처리 상태를 확인할 수 있다.
- 환불 사유, 환불 금액, 환불 처리 상태는 해당 티켓 타임라인에 저장된다.
- 긴급 수리에서 임대인 부담 가능성이 높은 경우 임대인 선결제 또는 자동결제 설정에 따라 처리될 수 있으며, 임차인에게는 결제 주체와 처리 상태만 표시한다.

안전결제 흐름 예시:

```
견적 확인
→ 결제 주체 확인
→ 임차인 또는 임대인 예치금 결제
→ 업체 수리 진행
→ 임차인 완료 확인 또는 재요청
→ 완료 확인 시 업체 지급
→ 취소/분쟁 시 환불 처리
```

### 20. 삭제 요청 처리 상태 및 비식별화 안내

- 임차인은 계약 종료 후 계약서 OCR, 사진, 민원 기록, 채팅 기록 등 본인 데이터 삭제를 요청할 수 있다.
- 삭제 요청 후 처리 상태를 확인할 수 있다.
- 삭제 요청 상태는 `접수`, `검토중`, `승인`, `일부 보관`, `반려`, `완료`로 구분한다.
- 정산, 분쟁, 회계 보관이 필요한 기록은 즉시 삭제되지 않을 수 있다.
- 보관이 필요한 기록은 삭제 대신 마스킹 또는 비식별화 처리될 수 있다.
- 임차인은 어떤 항목이 삭제되고, 어떤 항목이 보관되며, 보관 사유가 무엇인지 확인할 수 있다.
- 얼굴, 주소, 계좌, 우편물, 계약서 내 민감정보가 포함된 첨부파일은 마스킹본으로 전환될 수 있다.
- 삭제 또는 마스킹 처리 결과는 변경 이력으로 저장된다.

삭제 요청 상태 예시:

```
요청 상태: 일부 보관
삭제 완료: 일반 채팅 기록, 중복 사진
보관 대상: 정산 관련 민원 기록
보관 사유: 퇴실 정산 및 분쟁 대비
처리 방식: 원본 접근 제한 + 민감정보 마스킹
```

### 21. 업체 메시지 및 방문 일정 조율

- 업체가 배정된 티켓에서는 업체 메시지와 방문 일정이 해당 티켓 안에서 관리된다.
- 임차인은 업체명, 방문 예정 시간, 방문 상태, 준비 사항을 확인할 수 있다.
- 임차인은 방문 가능 시간대를 제안하거나 일정 변경 요청을 남길 수 있다.
- 방문 일정이 확정되면 임차인에게 앱 알림 또는 채팅 알림을 제공한다.
- 방문 시간이 변경되거나 지연되는 경우 변경 사유와 새 예정 시간을 표시한다.
- 업체와의 직접 연락처 노출은 최소화하고, 필요한 대화는 티켓 메시지로 남긴다.
- 업체 메시지, 일정 변경, 방문 완료 여부는 모두 티켓 타임라인에 저장된다.
- 방문 후 임차인은 완료 확인, 미해결 재요청, 만족도, 추가 사진을 남길 수 있다.

방문 일정 표시 예시:

```
방문 상태: 일정 확정
업체명: 가나다수리
방문 예정: 7월 1일 오후 2시~4시
임차인 확인: 가능
준비 사항: 화장실 천장 누수 부위 주변 정리
```
